# 욕망백서 탈퇴 / 데이터 삭제 정책 초안 v0.1

## 1. 목적

이 문서는 욕망백서 이용자의 탈퇴, 계정 종료, 데이터 삭제 및 보존 기준을 정리하기 위한 초안이다.

## 2. 기본 원칙

- 이용자는 탈퇴를 요청할 수 있다
- 탈퇴는 즉시 모든 기록을 완전히 제거하는 것과 동일하지 않을 수 있다
- 분쟁 대응, 부정 이용 방지, 법령 준수를 위해 일부 정보는 일정 기간 보관할 수 있다
- 사용자가 직접 작성한 공개 콘텐츠와 운영상 필요한 로그는 성격을 구분해서 처리한다

## 3. 탈퇴 시 처리되는 항목

### 즉시 또는 빠르게 비활성화되는 항목

- 로그인 및 서비스 접근 권한
- 프로필 공개
- DM 추가 이용
- 새로운 게시글 작성

### 삭제 또는 비노출 처리 대상

- 프로필 정보
- 자기소개
- 성향 정보
- 공개 게시글의 사용자 식별 정보

## 4. 보존이 필요한 항목

### 운영 및 분쟁 대응 목적

- 포인트 거래 이력
- 결제/입금 이벤트
- 신고 및 차단 이력
- 제재 이력
- 동의/상태 전이 로그

### 보존 이유

- 환불 및 결제 분쟁 대응
- 반복 부정 이용 방지
- 운영 정책 위반 이력 확인
- 법령상 의무 또는 정당한 운영상 필요

## 5. 메시지 데이터 기준

- 메시지 보관 기간은 현재 `90일` 운영 기준을 따른다
- 탈퇴한 계정의 메시지도 운영, 신고, 분쟁 대응 목적 범위에서 일정 기간 보관될 수 있다
- 다만 일반 사용자에게는 탈퇴 계정의 프로필 정보가 더 이상 노출되지 않도록 처리한다

## 6. 게시글 데이터 기준

- 탈퇴 시 공개 게시글은 삭제 또는 비노출 처리 기준을 정할 수 있다
- 운영상 필요한 경우 신고/분쟁과 연결된 게시글은 별도 보관할 수 있다
- 자동 소개글도 탈퇴 시 일반 노출 대상에서 제외한다

## 7. 포인트 / 결제 데이터 기준

- 포인트 거래 이력은 운영상 중요 기록으로 본다
- 결제, 환불, 수동 조정 이력은 탈퇴 이후에도 보관할 수 있다
- 남은 포인트의 소멸 및 환불 기준은 별도 정책 또는 운영 기준에 따른다

## 8. 차단 / 신고 / 제재 데이터 기준

- 차단 이력은 부정 이용 방지와 운영 참고를 위해 일정 기간 보관할 수 있다
- 신고 이력, 제재 이력, 운영 로그는 반복 위반 탐지 목적상 별도 보관할 수 있다

## 9. 탈퇴 후 사용자 화면 기준

- 탈퇴 완료 후 로그인 불가
- 프로필 비노출
- 기존 대화방에서는 상대 사용자에게 탈퇴 사실 또는 비활성 상태만 제한적으로 표시 가능

예시 문구:

- 이 계정은 더 이상 활성 상태가 아닙니다

## 10. 운영 처리 기준

### 탈퇴 요청 접수 시 확인할 것

- 남은 포인트 상태
- 결제/환불 분쟁 진행 여부
- 신고 또는 제재 이력
- 미처리 운영 이슈 존재 여부

### 운영 메모

- 분쟁 중인 계정은 일부 데이터 즉시 삭제보다 보존 판단이 우선일 수 있다
- 탈퇴와 제재는 다른 개념으로 구분한다

## 11. 제재 계정과의 구분

- 자발적 탈퇴 계정
- 운영상 `suspended` 또는 `banned` 계정

원칙:

- 탈퇴는 이용 종료 요청
- 제재는 운영 조치
- 두 경우 모두 데이터 보존 범위는 다를 수 있다

## 12. 추후 구체화 필요 항목

- 포인트 잔액 처리 기준
- 게시글 완전 삭제 vs 비노출 기준
- 탈퇴 후 재가입 제한 정책
- 법령상 보관 기간 세부 반영

## 13. 최종 판단

욕망백서의 탈퇴 정책은 `모든 것을 즉시 삭제한다`보다  
`무엇을 비노출하고, 무엇을 운영상 보존하는지`를 명확히 나누는 것이 중요하다.

그래야 사용자 안내와 운영 대응이 모두 일관될 수 있다.
