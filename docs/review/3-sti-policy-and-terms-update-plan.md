- 기준 문서
- [privacy-policy-v0.1.md](/Users/yeomj/OneDrive/Desktop/욕망백서/docs/legal/privacy-policy-v0.1.md)

- [operation-policy-v0.1.md](/Users/yeomj/OneDrive/Desktop/욕망백서/docs/legal/operation-policy-v0.1.md)
- 관련 법령
- 개인정보 보호법 제22조: 동의 사항 구분
- 개인정보 보호법 제23조: 민감정보 처리 제한

## 결론
- 현재 문서 상태로는 `최근 STI 검사 확인` 기능을 넣기 어렵다.
- 이유는 민감정보 처리, 공개 동의 분리, 파기 원칙, 철회 절차, 위조 제출 제재 기준이 문서상 분명하게 내려와 있지 않기 때문이다.
- 따라서 이 기능은 출시 전에 최소 4개 문서를 동시에 개정해야 한다.

## 1. 개인정보처리방침 반영안

### 반드시 추가할 항목
- 민감정보 처리 사실
- 처리 목적
- 처리 항목
- 보유 기간
- 공개 여부
- 파기 정책
- 이용자 권리 행사 방법

### 권장 문구 구조
1. 처리 목적
- `최근 STI 검사 확인 기능 제공`
- `제출 자료의 검수`
- `프로필 내 배지 표시`
- `이의제기 및 분쟁 처리`

2. 처리 항목
- 필수: 사용자 식별값, 검사일, 검증 상태, 유효기간, 검수 시각, 공개 여부
- 선택 또는 임시 처리: 검수용 제출 자료
- 저장 금지: 세부 질환 결과 공개본, 주민번호, 상세 병력

3. 보유 기간
- 공개용 메타데이터: 기능 이용 기간 또는 철회 시까지
- 검수용 원본: 검수 완료 후 즉시 삭제 원칙
- 감사 로그: 별도 보존 기간 명시

4. 권리 행사
- 공개 철회
- 삭제 요청
- 정정 요청
- 위조/오판정 이의제기

### 반영 포인트
- 기존 [privacy-policy-v0.1.md](/Users/yeomj/OneDrive/Desktop/욕망백서/docs/legal/privacy-policy-v0.1.md)에 일반 프로필/활동 데이터 중심 문구는 있으나, 민감정보 별도 처리 문단이 없습니다.
- 여기에는 `민감정보`라는 용어가 명시적으로 들어가야 합니다.

## 2. 이용약관 반영안

### 반드시 추가할 항목
- 기능 정의
- 회사 책임의 한계
- 이용자 진술 및 보증
- 허위 제출 금지
- 배지의 의미 제한

### 약관에 들어가야 할 핵심 원칙
- 배지는 `최근 검사 확인`일 뿐, 현재 상태 보증이 아니다.
- 회사는 이용자 간 접촉 이후 건강 상태를 보증하지 않는다.
- 허위 자료 제출 시 즉시 박탈 및 제재 가능
- 기능 이용은 선택 사항이며, 미이용자에게 기본 서비스 이용 제한을 두지 않는다.

### 권장 조항 예시
- `회사가 표시하는 배지는 최근 검사 확인 여부를 나타내는 정보이며, 현재의 건강 상태나 감염 가능성을 보증하지 않습니다.`
- `회원은 본인 자료만 제출하여야 하며, 허위 또는 타인 자료 제출 시 계정 제재를 받을 수 있습니다.`

### 반영 포인트
- 기존 [terms-of-service-v0.1.md](/Users/yeomj/OneDrive/Desktop/욕망백서/docs/legal/terms-of-service-v0.1.md)는 포인트, DM, 신고, 차단 중심입니다.
- 여기에 `민감정보 기반 신뢰 기능`에 대한 책임 한계가 별도로 필요합니다.

## 3. 운영정책 반영안

### 반드시 추가할 항목
- 검수 대상
- 검수 기준
- 위조 판단 기준
- 박탈 기준
- 재신청 기준
- 이의제기 처리

### 권장 정책
- 배지 부여 기준
- 최근 검사 확인이 가능한 자료만 허용
- 검수 기준일 초과 시 만료

- 박탈 기준
- 위조 자료
- 타인 자료
- 검수 후 허위 사실 확인
- 이용자 공개 철회

- 제재 기준
- 1회 허위 제출: 즉시 박탈 + 경고 또는 일정 기간 제한
- 반복 위조: 계정 제한 또는 영구 정지 검토

### 반영 포인트
- 기존 [operation-policy-v0.1.md](/Users/yeomj/OneDrive/Desktop/욕망백서/docs/legal/operation-policy-v0.1.md)는 신고/차단/금지 행위 중심입니다.
- 여기에 `민감정보 검수 기능 운영` 장이 새로 필요합니다.

## 4. 별도 동의서 필요 여부

### 결론
- 필요합니다.

### 최소 분리 단위
1. 민감정보 수집·이용 동의
2. 검수 목적 처리 동의
3. 프로필 공개 동의

### 이유
- 개인정보 보호법 제22조는 동의 사항을 구분해 명확히 인지하도록 요구합니다.
- 개인정보 보호법 제23조 관련 동의는 특히 일반 동의에 섞지 않는 것이 안전합니다.
