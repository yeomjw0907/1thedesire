# STI 기능 수익모델 연결 검토안

## 목적
- `최근 STI 검사 확인` 기능을 서비스 수익모델과 어떻게 연결할지 검토한다.
- 기준은 `신뢰 강화`, `전환 개선`, `법무 리스크 증가 최소화`다.

## 결론
- 이 기능은 직접 과금 기능보다 `신뢰 인프라`로 보는 게 맞다.
- 즉시 유료화하면 거부감과 법무 리스크가 같이 올라간다.
- 1차 역할은 `DM 전환율 개선`, `신고율 감소`, `고가치 유저 유지`다.

## 1. 가능한 포지셔닝

### 포지셔닝 A. 무료 신뢰 기능
- 장점: 진입장벽 낮음, 브랜드 신뢰 강화
- 단점: 운영비 회수 어려움

### 포지셔닝 B. 프리미엄 프로필 강화 요소
- 장점: 수익 연결 가능
- 단점: 민감정보 기능 유료화에 대한 거부감 가능

### 포지셔닝 C. 유료 기능이 아니라 유료 전환 보조 장치
- 장점: 가장 현실적
- 설명: 배지 자체는 무료 또는 제한 무료, 대신 이 배지로 DM 과금 전환율과 프리미엄 구독 전환을 올리는 구조

## 2. 추천 포지셔닝

### 권장안
- `유료 기능이 아닌 신뢰 강화 기능`
- 수익화는 간접 방식

### 이유
- 민감정보 기능을 직접 팔면 불신을 부를 수 있다.
- 반면 신뢰 신호는 DM 과금과 오프라인 만남 전환의 저항을 줄일 수 있다.

## 3. 연결 가능한 KPI
- 프로필 조회 대비 DM 요청 전환율
- DM 결제 전환율
- 신고율
- 차단율
- 2주/4주 유지율
- 고의도 유저 재방문율

## 4. 간접 수익화 시나리오

### 시나리오 1. DM 전환 강화
- 배지 보유 프로필에서 DM 요청률 상승 유도
- 결과적으로 포인트 사용량 증가 가능

### 시나리오 2. 프리미엄 신뢰 프로필 패키지
- 배지 자체는 무료
- 대신 `신뢰 정보 섹션 확장`, `상세 소개 강화`, `우선 노출` 등을 프리미엄으로 묶을 수 있음

### 시나리오 3. 오프라인 만남 안전 플로우 확장
- 검사 확인
- 경계선 공유
- 보호수단 선호
- 만남 체크리스트

### 설명
- 이 묶음이 쌓이면 프리미엄 가치 제안이 만들어진다.

## 5. 비권장 수익화
- 검사 확인 배지 단독 유료 판매
- 빠른 승인 유료 판매
- 배지 미보유자 불이익 노출
- 높은 등급의 건강 점수화

## 6. 예상 비용 요소
- 운영자 검수 인건비
- 스토리지 및 보안 비용
- 고객 문의 대응
- 법무/자문 비용

## 7. 수익모델 적합성 판단 기준

### 출시할 가치가 있는 경우
- DM 전환율 개선이 확인됨
- 신고/분쟁 감소 효과가 있음
- 고가치 유저 유지율이 올라감

### 재검토해야 하는 경우
- 제출률이 매우 낮음
- 운영비가 과도함
- 오해/민원/법적 우려가 큼

## 8. 단계별 전략 권장안

### 1단계
- 무료 신뢰 기능으로 도입
- 전환율과 운영비 측정

### 2단계
- 프리미엄 프로필 패키지의 일부로 연결 검토

### 3단계
- 제휴형 확장 또는 안전 기능 묶음 상품화 검토

## 9. 권장 결론
- 이 기능은 단기 매출 기능보다 장기 신뢰 자산으로 다뤄야 한다.
- 직접 과금보다 간접 전환 효과를 KPI로 두는 편이 맞다.
