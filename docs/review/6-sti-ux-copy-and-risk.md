# STI 기능 UX 문구 및 노출 리스크 검토안

## 목적
- `최근 STI 검사 확인` 기능이 사용자에게 어떻게 보여야 하는지, 어떤 문구가 오해를 줄이고 신뢰를 높이는지 정리한다.

## 결론
- UX의 핵심은 `신뢰 신호`를 주되 `안전 보장`처럼 보이지 않게 만드는 것이다.
- 배지 자체보다 문구, 만료 표시, 클릭 시 고지 문구가 더 중요하다.

## 1. 배지 노출 원칙

### 보여줄 정보
- `최근 STI 검사 확인`
- `검사일 YYYY-MM-DD`
- `유효기간 YYYY-MM-DD까지`

### 보여주지 말아야 할 정보
- 세부 질환 리스트
- 음성/양성 판정 텍스트
- 병원 원본 이미지
- 과도한 의학적 표현

## 2. 가장 중요한 UX 리스크

### 리스크 1. 사용자가 `현재 안전`으로 오해
- 원인: 배지 명칭이 강할수록 발생
- 대응: `검사 시점 기준`, `현재 상태 보증 아님` 고정 문구 노출

### 리스크 2. 배지 미보유자 낙인
- 원인: 배지 없는 유저를 위험하게 보이게 만드는 레이아웃
- 대응: 미보유자는 `표시 없음`으로만 처리하고 경고성 문구를 붙이지 않음

### 리스크 3. 위계화된 매칭 경험
- 원인: 배지를 지나치게 상단 강조하면 다른 정보가 묻힘
- 대응: 프로필의 신뢰 정보 모듈 중 하나로 배치

## 3. 권장 노출 위치

### 프로필 상세
- 닉네임/기본 정보 아래
- 신뢰 정보 섹션 내부
- 클릭 시 설명 모달

### DM 요청 전 바텀시트
- 상대의 최근 검사 확인 여부 노출
- 동시에 경고 문구 노출

### 검색/리스트
- 칩 형태로 최소 노출
- 리스트에서는 과도한 강조 금지

## 4. 권장 카피

### 배지 기본 문구
- `최근 STI 검사 확인`

### 보조 문구
- `검사일 2026-03-01`
- `2026-04-01까지 표시`

### 상세 설명 문구
- `본 표시는 검사 시점 기준 정보입니다. 현재의 건강 상태를 보증하지 않으며, 만남 전 추가 확인과 보호수단 사용을 권장합니다.`

### CTA 문구
- `검사 확인 정보 보기`
- `검사일 확인`

## 5. 금지 카피
- `안전 인증`
- `무감염 인증`
- `안심 프로필`
- `성병 없음 확정`
- `믿고 만나도 됨`

## 6. 사용 흐름별 권장 UX

### 제출 전
- 이 기능이 무엇인지 설명
- 어떤 정보가 공개되는지 명확히 표시
- 철회 가능성 안내

### 제출 후 대기
- `검수 중`
- 예상 처리 시간
- 공개 전이라는 점 안내

### 승인 후
- 배지 노출
- 만료일 안내
- 철회 버튼 제공

### 반려 후
- 반려 사유 간단 명시
- 재제출 안내

## 7. 배지 디자인 원칙
- 의료기관 인증처럼 과도한 권위 배지 스타일 금지
- `메타 인증 마크`처럼 절대적 상징으로 보이는 디자인은 피함
- 중립적 칩/배지 형태
- 녹색 일변도보다 중립 색상 기반 + 정보성 강조

## 8. 추가 신뢰 UX 권장안
- `보호수단 선호`
- `만남 전 대화 선호`
- `경계선 공유 여부`
- `최근 활동`

### 이유
- 검사 배지 하나만으로 신뢰를 설명하면 과잉 해석된다.
- 신뢰는 여러 신호의 조합으로 보이게 해야 한다.

## 9. 접근성 포인트
- 배지 색만으로 상태 전달 금지
- 만료 상태는 텍스트로 명확히 표기
- 설명 모달은 스크린리더 친화 문구 사용

## 10. 권장 결론
- 배지는 강한 세일즈 장치보다 차분한 신뢰 정보가 맞다.
- 과도한 강조는 법적 리스크와 UX 오해를 동시에 키운다.
