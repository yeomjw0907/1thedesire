# STI 검사 확인 기능 법적 리스크 최소화 정책안 초안

## 목적
- 본 문서는 서비스 내 `성병 12종 없음 인증`과 유사한 기능을 검토할 때, 한국 법 체계 안에서 운영 가능한 최소 범위를 정의하기 위한 초안이다.
- 결론부터 말하면, 서비스는 `무감염 보증`이 아니라 `최근 STI 검사 확인`만 다뤄야 한다.

## 전제
- 검사 결과, 건강 상태, 성생활 관련 정보는 민감정보로 취급해야 한다.
- 따라서 일반 프로필 정보와 같은 수준으로 수집, 저장, 공개하면 안 된다.
- 이 기능은 마케팅 기능이 아니라 `민감정보 처리 + 공개 통제 + 표시 책임` 기능으로 본다.

## 1. 허용 범위

### 허용 가능한 기능
- 사용자 신청 기반의 `최근 STI 검사 확인 배지`
- 공개 정보는 아래 항목으로 제한
- `검사 확인 여부`
- `검사일`
- `유효기간 만료일`
- `검증 완료 시각`
- `검증 방식 코드` 예: 수동검수, 제휴기관 확인

### 허용하지 않는 기능
- `성병 없음`, `안전 인증`, `안심 상대`, `무감염 보장` 문구
- 세부 검사 결과 항목 전체 공개
- 양성/음성 세부 판정 공개
- 원본 검사 결과지 장기 저장
- 배지 미보유자를 불이익 집단처럼 보이게 하는 UI
- 사용자 동의 없는 자동 공개

## 2. 핵심 법적 리스크

### 1. 민감정보 처리 리스크
- 건강 관련 정보는 민감정보로 보아야 한다.
- 민감정보는 법령 근거가 있거나 별도 동의가 있는 경우 등에만 처리 가능하다.
- 서비스는 반드시 `민감정보 처리 별도 동의`를 받아야 한다.

### 2. 민감정보 공개 리스크
- 프로필 배지는 사실상 제3자 공개 기능이다.
- 따라서 `수집/검수 동의`와 `프로필 공개 동의`를 분리해야 한다.
- 사용자는 언제든 공개 철회와 배지 비노출을 선택할 수 있어야 한다.

### 3. 허위·과장·기만 표시 리스크
- 검사 시점 이후 상태까지 안전하다고 읽히면 표시광고법 리스크가 생긴다.
- 특히 `안전`, `무감염`, `보장` 문구는 피해야 한다.
- 기능 설명과 배지 문구 모두 `검사 시점 기준`임을 명시해야 한다.

### 4. 보관 과다 리스크
- 원본 결과지를 계속 보관하면 민감정보 유출 리스크가 커진다.
- 목적 달성 후에는 지체 없이 파기해야 한다.
- 검수 완료 후 원본 삭제, 최소 메타데이터만 잔존이 기본 원칙이다.

### 5. 국외이전 리스크
- 인프라나 검수 조직이 해외에 있으면 국외이전 이슈가 붙을 수 있다.
- 사용 인프라와 운영 주체의 위치를 먼저 확정해야 한다.
- 해외 저장이나 열람이 개입되면 별도 고지와 이전 요건 검토가 필요하다.

## 3. 최소화 정책

### 정책 1. 기능명 제한
- 대외 명칭은 `최근 STI 검사 확인`
- 내부 코드명도 `sti_verification` 대신 `recent_sti_check_confirmation`처럼 보증 뉘앙스가 없는 표현 권장

### 정책 2. 수집 최소화
- 저장 가능 항목
- 사용자 ID
- 검증 상태
- 검사일
- 만료일
- 검수 완료 시각
- 검수 방식
- 철회 여부
- 운영 로그

- 저장 금지 항목
- 검사 결과 원문 전문
- 병원 문서 이미지 장기 보관
- 주민등록번호, 주소, 상세 병력
- 세부 질환별 결과의 공개용 저장

### 정책 3. 원본 파기
- 원본 제출물이 필요한 경우에도 검수 완료 후 즉시 삭제를 원칙으로 한다.
- 예외적으로 분쟁 대응 보존이 필요하면 최소 기간만 별도 암호화 보관하고, 운영자가 임의 열람하지 못하게 접근통제를 둔다.
- 기본안은 `원본 비보관`이다.

### 정책 4. 동의 분리
- 동의는 최소 3개로 분리
- `민감정보 수집·이용 동의`
- `검수 목적 처리 동의`
- `프로필 공개 동의`

- 공개 동의는 선택 동의여야 한다.
- 동의 철회 즉시 배지는 비노출 처리되어야 한다.

### 정책 5. 공개 제한
- 공개 화면에는 세부 결과 대신 아래만 노출
- `최근 STI 검사 확인`
- `검사일 YYYY-MM-DD`
- `유효기간 YYYY-MM-DD까지`

- 사용자 클릭 시 안내 문구 노출
- `본 표시는 검사 시점 기준 정보이며 현재의 건강 상태를 보증하지 않습니다.`

### 정책 6. 유효기간 강제
- 배지는 영구 배지 금지
- 만료일 지나면 자동 숨김
- 권장안
- 기본 유효기간 30일 또는 60일
- 만료 후 자동 재검증 필요

### 정책 7. 운영 권한 제한
- 검수 권한은 최소 인원에게만 부여
- 접근 로그, 조회 로그, 변경 로그를 모두 남김
- 운영자 화면에서도 원본 열람은 기본 금지
- 가능하면 검수 전용 별도 저장소/별도 관리자 권한 체계 사용

### 정책 8. 분쟁 대응
- 사용자는 배지 철회, 오류 수정, 삭제 요청 가능
- 위조 제출, 허위 제출 발견 시 즉시 배지 박탈
- 반복 위반 시 계정 제재 근거를 운영정책에 명시

## 4. 추천 데이터 구조


- 프로필 테이블에 민감정보 칼럼 직접 추가
- 공개용 API에서 세부 질환 결과 반환

## 5. 서비스 문구 초안

### 배지 문구
- `최근 STI 검사 확인`
- `검사일 2026-02-10`
- `2026-03-12까지 표시`

### 보조 문구
- `검사 시점 기준으로 확인된 정보입니다.`
- `현재 상태를 보증하지 않으며, 만남 전 추가 확인과 보호수단 사용을 권장합니다.`

### 금지 문구
- `안전 인증`
- `성병 없음 인증`
- `무감염 인증`
- `안심 상대`
- `100% 안전`

## 6. 동의 문구 초안

### 민감정보 수집·이용 동의 초안
`회사는 최근 STI 검사 확인 기능 제공을 위해 이용자가 제출한 건강 관련 정보를 민감정보로서 처리합니다. 처리 항목은 검사 확인을 위한 최소 정보에 한정되며, 검증 목적 외로 이용하지 않습니다. 귀하는 동의를 거부할 권리가 있으나, 이 경우 해당 기능을 이용할 수 없습니다.`

### 공개 동의 초안
`회사는 귀하의 선택에 따라 최근 STI 검사 확인 여부, 검사일, 유효기간 정보를 귀하의 프로필에 공개할 수 있습니다. 귀하는 언제든 공개 동의를 철회할 수 있으며, 철회 시 배지는 즉시 비공개 처리됩니다.`

### 고지 문구 초안
`본 배지는 검사 시점 기준으로 최근 검사 확인 여부를 표시하는 기능입니다. 회사는 현재의 건강 상태나 향후 감염 가능성을 보증하지 않습니다.`

## 7. 운영 정책 초안

### 배지 부여 기준
- 회사가 정한 방식으로 최근 검사 사실이 확인된 경우에만 부여
- 기준일 경과 시 자동 만료

### 박탈 기준
- 위조 자료 제출
- 타인 자료 제출
- 검증 후 허위 사실 발견
- 이용자 본인 철회

### 이용자 권리
- 공개 여부 선택
- 철회 요청
- 삭제 요청
- 오류 정정 요청

## 8. 출시 전 체크리스트

- 별도 동의서 분리 작성
- 운영정책과 제재 기준 반영
- 원본 저장 여부 확정
- 저장소 지역과 운영자 접근 권한 확정
- 유효기간 자동 만료 로직 구현
- 배지 공개/비공개 철회 UX 구현
- 보증 금지 문구 전 화면 반영
- 침해사고 대응 프로세스 준비
