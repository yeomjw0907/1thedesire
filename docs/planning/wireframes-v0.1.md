# 욕망백서 와이어프레임 문서 v0.1

## 1. 목적

- 홈, 프로필, DM 핵심 화면의 정보 구조를 빠르게 정리한다.
- 디자인 이전에 화면이 전달해야 하는 정보와 행동 순서를 고정한다.

## 2. 공통 원칙

- 모바일 한 손 사용 기준
- 자극보다 신뢰 우선
- 과금과 상태는 숨기지 않음
- 프로필보다 피드, 피드보다 분위기를 먼저 읽게 함

## 3. 홈 화면

### 3-1. 목적

- 유저가 서비스 분위기와 사람의 결을 빠르게 읽는다.
- 게시글에서 프로필로 자연스럽게 이동한다.

### 3-2. 구조

```text
[상단 바]
- 로고
- 검색 진입
- 포인트 잔액

[필터 탭]
- 전체
- 신규

[피드 리스트]
- 게시글 카드 반복

[플로팅 버튼]
- 글쓰기

[하단 탭]
- 홈
- 검색
- DM
- 포인트
- 프로필
```

### 3-3. 게시글 카드 구조

```text
[작성자 요약]
- 닉네임
- 연령대 / 지역
- 성향 태그 일부
- 작성 시간

[본문]
- 텍스트 2~4줄 미리보기
- 이미지 1장 썸네일 가능

[하단 액션]
- 프로필 보기
- 신고
```

### 3-4. 홈 UX 포인트

- 첫 화면에서 너무 많은 필터를 주지 않는다.
- 게시글이 곧 사람의 분위기라는 느낌을 줘야 한다.
- 포인트보다는 콘텐츠를 먼저 보게 한다.

## 4. 프로필 화면

### 4-1. 목적

- DM을 보낼지 말지 판단하게 한다.
- 외모보다 취향과 분위기를 먼저 읽게 한다.

### 4-2. 구조

```text
[상단]
- 뒤로가기
- 신고
- 차단

[프로필 헤더]
- 닉네임
- 연령대 / 지역
- 성별
- 성향

[신뢰/상태 영역]
- 최근 활동
- DM 가능 상태
- 여성일 경우: 받은 DM 수락 무료 안내

[자기소개]
- 본문 전체 노출

[최근 게시글]
- 최근 글 3~5개 미리보기

[고정 하단 액션]
- DM 요청 버튼
- 90P 차감 / 거절 50% 환불 / 미응답 전액 환불 안내
```

### 4-3. 프로필 UX 포인트

- 사진보다 정보가 먼저 보여야 한다.
- DM 버튼은 강하되 부담스럽지 않아야 한다.
- 과금 규칙은 버튼 아래에서 짧고 분명하게 보여준다.
- 신고/차단은 숨기지 않되 메인 액션을 침범하면 안 된다.

## 5. DM 요청 팝업

### 5-1. 목적

- 과금과 요청 상태를 명확히 이해시키고 요청을 확정한다.

### 5-2. 구조

```text
[팝업 제목]
- 대화 요청 보내기

[상대 요약]
- 닉네임
- 연령대 / 지역
- 성향

[정책 요약]
- 요청 시 90P 차감
- 거절 시 45P 환불
- 24시간 내 미응답 시 전액 환불
- 수락 후 대화는 무료

[액션]
- 취소
- 요청 보내기
```

### 5-3. UX 포인트

- 법적 표현보다 서비스 규칙을 먼저 보여준다.
- 사용자가 손해보는 구조처럼 느끼지 않게 환불 규칙을 함께 보여준다.

## 6. DM 목록 화면

### 6-1. 목적

- 요청 대기와 진행 중 대화를 한눈에 관리한다.

### 6-2. 구조

```text
[상단]
- DM

[탭]
- 대기
- 진행중
- 종료

[리스트 카드]
- 상대 닉네임
- 상태 배지
- 요청 시각 또는 마지막 메시지 시각
- 환불 여부 요약
```

### 6-3. 상태별 카드 예시

- `pending`: 응답 대기중
- `agreed`: 대화 가능
- `declined`: 45P 환불 완료
- `expired`: 90P 환불 완료
- `blocked`: 대화 불가

## 7. 채팅방 화면

### 7-1. 목적

- 수락 이후 대화를 자연스럽게 이어간다.
- 차단과 신고는 필요할 때 빠르게 접근 가능해야 한다.

### 7-2. 구조

```text
[상단]
- 뒤로가기
- 상대 닉네임
- 프로필 보기
- 신고
- 차단

[상태 배너]
- pending: 상대 응답을 기다리는 중입니다
- agreed: 대화를 시작할 수 있습니다
- declined: 요청이 거절되어 45P 환불되었습니다
- expired: 요청이 만료되어 90P 환불되었습니다
- blocked: 차단되어 대화할 수 없습니다

[메시지 리스트]
- 시스템 메시지
- 사용자 메시지

[입력 영역]
- pending/declined/expired/blocked: 비활성
- agreed: 활성
```

### 7-3. 채팅 UX 포인트

- 상태가 분명해야 한다.
- 입력 불가 이유를 감추지 않는다.
- 환불이 일어난 경우 바로 체감되게 보여준다.

## 8. 화면 간 연결 흐름

```text
홈 피드
-> 게시글 클릭
-> 프로필
-> DM 요청 팝업
-> DM 목록
-> 채팅방
```

## 9. 우선 제작 순서

1. 홈
2. 프로필
3. DM 요청 팝업
4. DM 목록
5. 채팅방

## 10. 최종 판단

욕망백서의 핵심 화면은 모두 같은 흐름을 만들어야 한다.

- 먼저 분위기를 읽고
- 다음으로 사람을 판단하고
- 그 다음에 요청하고
- 마지막에 대화를 시작한다

이 순서가 흔들리면 서비스가 그냥 성인 채팅앱처럼 보이게 된다.
