# 욕망백서 프론트엔드 구현 체크리스트 v0.1

## 1. 목적

- 프론트엔드 구현 시 필요한 화면, 상태, 문구, UX 기준을 체크리스트로 정리한다.
- 다음 작업자가 문서를 훑지 않고도 우선순위대로 구현할 수 있게 한다.

## 2. 공통 기준

- 모바일 우선
- PWA 우선
- 홈은 탐색
- 프로필은 판단
- DM은 요청 절차
- 과금/환불 상태는 숨기지 않음
- 브랜드 톤은 자극보다 신뢰

## 3. 앱 셸

- 상단 바 구조 확정
- 하단 탭 바 `홈 / 검색 / DM / 포인트 / 프로필`
- 포인트 잔액 표시 위치 확정
- 로딩 / 빈 상태 / 에러 상태 UI 준비

## 4. 인증 / 가입

- 로그인 시작 화면
- 간편 로그인 화면
- 가입 폼 화면
- 19세 이상 체크
- 약관 동의 체크
- 자기소개 입력 필수 처리
- 가입 완료 화면
- 여성 270P 지급 안내 노출
- 자동 소개글 안내 노출

## 5. 홈 피드

- 홈 피드 목록 화면
- `전체 / 신규` 탭
- 게시글 카드
- 프로필 이동
- 신고 액션
- 글쓰기 플로팅 버튼
- 빈 상태 문구 반영

## 6. 글쓰기

- 게시글 작성 화면
- 이미지 1장 업로드 UI
- 게시 완료
- 삭제 기능
- 수정 진입 가능 여부 정리

## 7. 프로필

- 프로필 상세 화면
- 닉네임 / 연령대 / 지역 / 성별 / 성향 표시
- 자기소개
- 최근 게시글 리스트
- 최근 활동 / DM 가능 상태 영역
- 여성 대상 `받은 요청 수락 무료` 보조 문구
- 신고 / 차단 버튼
- 하단 고정 DM CTA

## 8. 검색 / 탐색

- 검색 화면
- 검색창
- 빠른 필터 `지역 / 성향 / 태그`
- 결과 카드
- 프로필 카드 / 게시글 카드

## 9. DM 요청

- DM 요청 팝업 또는 바텀시트
- 상대 요약 정보
- 정책 패널
- `90P 차감`
- `거절 시 45P 환불`
- `24시간 미응답 시 전액 환불`
- `수락 후 무료 대화`
- 취소 / 요청 보내기 버튼

## 10. DM 목록

- `대기 / 진행중 / 종료` 탭
- 상태 카드
- 상태별 문구 노출
- 환불 여부 노출

## 11. 채팅방

- 채팅방 상단 바
- 프로필 보기
- 신고 / 차단
- 상태 배너
- 메시지 리스트
- 입력창 활성 / 비활성 분기
- 시스템 메시지 스타일

## 12. 포인트 화면

- 현재 잔액 카드
- 충전 안내
- 거래 내역 탭
- 차감 / 환불 / 충전 구분
- 여성 메리트 설명 카드

## 13. 신고 / 차단 UI

- 신고 바텀시트
- 차단 확인 시트
- 완료 토스트
- 상태 문구 반영

## 14. 알림 / 재방문 UX

- 인앱 알림 화면 또는 알림 센터
- DM 상태 변화 우선 노출
- 환불 알림 반영
- 첫 7일 리텐션 문구 반영

## 15. 카피 / 브랜드 점검

- 카피 라이브러리 기준 반영
- 과장형 문구 제거
- 저가형 성인앱 느낌 제거
- 20대 후반~30대 후반 톤 유지

## 16. 프론트엔드 구현 순서

1. App Shell
2. 가입 / 온보딩
3. 홈 피드
4. 프로필
5. DM 요청 / 목록 / 채팅
6. 포인트
7. 신고 / 차단
8. 검색 / 알림

## 17. 최종 체크

- 상태 문구가 백엔드 상태값과 맞는가
- 포인트 숫자가 정책 문서와 맞는가
- 화면 톤이 디자인 시스템과 맞는가
- 문구가 카피 라이브러리와 맞는가

## 18. 최종 판단

프론트엔드 구현의 핵심은 `예쁜 화면`이 아니라  
`상태가 분명하고, 신뢰가 무너지지 않는 화면`을 만드는 것이다.
