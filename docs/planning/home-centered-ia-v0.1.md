# 홈 중심 IA 개편안 v0.1

## 1. 목적

- 서비스의 메인 탐색 축을 `검색`이 아니라 `홈 게시글 피드`로 재정의한다.
- 사용 흐름을 `홈 -> 프로필 -> DM`으로 단순화한다.
- 중복 페이지를 줄이고, 게시글 중심의 체류와 DM 전환을 강화한다.

## 2. 핵심 결정

1. `홈`은 게시글 메인 피드이자 탐색 허브로 사용한다.
2. 기존 `검색` 기능은 홈의 필터/정렬/탐색 기능으로 편입한다.
3. 별도 `검색` 페이지는 제거 대상으로 본다.
4. 게시글 카드의 `프로필 보기` 버튼은 제거하고, 프로필 이미지 또는 닉네임 클릭으로 프로필에 진입한다.
5. 게시글 카드에는 `DM 보내기` 액션을 유지한다.
6. `DM 거절` 기능은 제거한다.
7. `DM` 목록 구조는 `전체 / 읽지 않음 / 요청`으로 재편한다.
8. `프로필`은 판단 페이지로 강화하며, 프로필 이미지, 상세 성향, 최근검사 확인, 신뢰 정보를 포함한다.

## 3. 최종 IA

```text
루트
-> 로그인 / 가입
-> 홈
-> DM
-> 포인트
-> 프로필
-> 관리자
```

## 4. 화면별 역할

### 4-1. 홈

- 메인 게시글 피드
- 탐색, 필터, 정렬, 카테고리 통합
- 게시글 작성 진입
- 프로필 진입
- DM 전환

### 4-2. DM

- 내가 보낸 DM과 받은 DM을 한 곳에서 관리
- 요청/읽지 않음/전체 중심으로 정리
- 읽음 여부와 요청 상태 확인

### 4-3. 포인트

- 충전
- 거래내역 확인
- 충전/환불/오입금 안내

### 4-4. 프로필

- 프로필 이미지
- 기본 정보
- 상세 성향
- 최근검사 확인
- 최근 게시글
- DM 전환

### 4-5. 관리자

- 신고/차단/제재
- 게시글/유저 관리
- 포인트 충전 관리
- 최근검사 확인 검수

## 5. 제거 또는 축소 대상

### 5-1. 검색 페이지

- 독립 페이지로 유지하지 않는다.
- 홈 필터가 자리 잡으면 제거한다.

### 5-2. DM 거절 상태

- 사용자 액션에서 제거한다.
- 관련 상태, 환불 문구, 운영 문서도 함께 정리 대상이다.

## 6. 사용자 핵심 흐름

```text
로그인
-> 홈에서 게시글 탐색
-> 프로필 이미지/닉네임 클릭
-> 프로필에서 상세 판단
-> DM 보내기
-> DM 목록에서 요청/읽지 않음/전체 관리
```

## 7. 우선순위

1. 홈을 게시글 메인 + 필터 허브로 개편
2. DM 구조를 `전체 / 읽지 않음 / 요청`으로 변경
3. 프로필 강화
4. 포인트 화면 실전형 정리
5. 검색 페이지 제거

## 8. 메모

- 홈이 강해질수록 서비스의 정체성이 선명해진다.
- 검색은 기능일 뿐, 메인 진입축이 아니어야 한다.
- DM은 운영 상태보다 사용자의 행동 기준으로 보이게 해야 한다.
