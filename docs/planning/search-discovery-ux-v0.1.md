# 욕망백서 검색 / 탐색 UX 문서 v0.1

## 1. 목적

- 검색과 탐색을 단순 조회 기능이 아니라 `사람의 결을 찾는 경험`으로 설계한다.
- 홈 피드와 프로필 중심 구조를 보완하는 탐색 레이어를 정리한다.

## 2. 검색과 탐색의 차이

### 검색

- 이미 어느 정도 조건이 있는 상태에서 찾는 행위

예:

- 지역으로 찾기
- 닉네임으로 찾기
- 성향으로 찾기

### 탐색

- 아직 정확한 조건보다 분위기와 결을 찾는 행위

예:

- 최근 올라온 글 읽기
- 특정 태그 흐름 보기
- 프로필 둘러보기

욕망백서에서는 탐색이 검색보다 먼저다.

## 3. 탐색 UX의 핵심 원칙

- 사람을 숫자보다 분위기로 먼저 찾게 한다
- 필터는 깊게 들어가기 전까지 단순해야 한다
- 결과는 카드보다 `관심 가는 결`로 보여야 한다
- 검색 화면도 딱딱한 데이터베이스처럼 보이면 안 된다

## 4. 검색 화면 구조

```text
[상단 검색창]

[빠른 필터]
지역 / 성향 / 태그

[추천 탐색 섹션]
요즘 많이 보이는 태그
최근 활발한 분위기

[결과 리스트]
프로필 카드 또는 게시글 카드
```

## 5. 검색창 설계

### 역할

- 닉네임 또는 키워드 기반 빠른 진입

### placeholder 후보

- 닉네임, 지역, 성향으로 찾아보세요
- 어떤 분위기를 찾고 있나요

### 원칙

- 검색창 하나만 던져놓지 않기
- 아래에 탐색 가이드를 같이 둔다

## 6. 빠른 필터 설계

### 초기 필터

- 지역
- 성향
- 태그

### 원칙

- 첫 진입부터 필터가 너무 많으면 안 됨
- 클릭 한 번으로 풀리는 정도의 가벼운 구조

## 7. 결과 카드 방향

### 프로필 결과 카드

구성:

- 닉네임
- 연령대 / 지역
- 성향
- 자기소개 일부
- 최근 글 한 줄

원칙:

- 사진보다 정보 우선
- 프로필 진입을 유도

### 게시글 결과 카드

구성:

- 작성자
- 본문 일부
- 태그
- 시간

원칙:

- 피드 흐름을 해치지 않도록 홈 카드와 유사한 톤 유지

## 8. 추천 탐색 섹션

### 목적

- 검색을 안 해도 둘러보게 만든다

예시 섹션:

- 요즘 많이 보이는 태그
- 최근 올라온 분위기
- 조용한 대화를 선호하는 사람들

원칙:

- 지나치게 선정적이거나 자극적인 분류 금지
- 서비스의 결을 보여주는 큐레이션 중심

## 9. 검색 / 탐색에서 피해야 할 것

- 과도한 정렬/필터 기능
- 랭킹 중심 노출
- 인기순만 강조하는 구조
- 즉시 만남 느낌의 카드 디자인
- 얼굴 사진 중심 배열

## 10. UX 문구 방향

### 추천 문구

- 어떤 분위기를 찾고 있나요
- 사람을 찾기보다 결을 찾아보세요
- 프로필보다 먼저 글의 온도를 읽어보세요

### 비추천 문구

- 지금 가능한 사람 찾기
- 핫한 회원 검색
- 즉시 만남 검색

## 11. 성공 기준

- 검색 없이도 탐색이 자연스럽다
- 사용자가 사람을 숫자보다 분위기로 기억한다
- 프로필 진입률이 높다
- 검색 화면이 차갑지 않다

## 12. 최종 판단

욕망백서의 검색은 `데이터 필터`가 아니라  
`대화를 시작하기 전 사람의 결을 더 정확히 좁혀가는 과정`이어야 한다.

그래서 탐색은 항상 검색보다 앞에 있어야 한다.
