# 성향(role) BDSM UX 계획 v0.1

시장 조사와 UX 관점 정리 후, 현재 구현과의 정합성 및 보완 사항을 고정한 문서입니다.

---

## 1. 시장 조사 요약

### 1.1 Feeld
- **Desires**: 30개 이상, 7개 카테고리(키워드·판타지 포함). 커스텀 입력 없음, 큐레이션만 제공.
- **글로서리**: 프로필/필터/온보딩에서 **탭 앤 홀드**로 용어 정의 카드 표시. 30+ desires, 20+ sexualities 등 정의 제공.
- **온보딩**: 성별·성적지향·desires·관계 선호를 한 번에 수집. “낯선 용어는 길게 눌러 확인” 안내.

### 1.2 FetLife
- 역할·성별·성적지향·관계 유형 **복수 선택** 지원.
- “맞는 태그가 없으면 제안 보내기”로 커스텀 수요 수용.

### 1.3 BDSM 데이팅 앱 (KinkS, KINK People, Switched 등)
- **단일 스케일(Active↔Passive)은 구식**. 다중 역할(Dom, Sub, Switch, Master, Mistress 등) 선택이 표준.
- **시각적 요약**: “한눈에 보이는 시각 표현” 강조. 칩/태그로 여러 개 선택 후 프로필에 태그 형태로 노출.

### 1.4 칩/다중 선택 UX (일반)
- **칩**: 체크박스·드롭다운보다 인지 부담 적고, 10~15개 수준 옵션에 적합. **선택 상태는 배경/테두리로 명확히**.
- **터치 타겟**: 최소 **44~48px** 높이 권장(a11y).
- **선택 개수**: 소규모 옵션은 “선택된 항목을 태그로 계속 노출”하는 패턴이 일반적.

## 2. 우리 서비스와의 정합성

| 항목 | 시장 관행 | 현재 구현 | 판단 |
|------|-----------|-----------|------|
| 다중 선택 | 복수 역할/성향 표준 | 칩 토글 + ` · `로 결합 저장 | 적합 |
| 그룹화 | Feeld 카테고리, 역할/취향 구분 | 역할·포지션 / 취향·플레이 / 관계·스타일 | 적합 |
| 선택 피드백 | 선택 칩 시각적 구분 | desire 색 강조 | 적합 |
| 직접 입력 | Feeld 없음, FetLife는 제안 수용 | 텍스트 필드 + 칩 병행 | 소규모·한국어 서비스에 유지 권장 |
| 용어 설명 | Feeld 탭 앤 홀드 글로서리 | 없음 | 보완 대상 |
| 터치 타겟 | 44~48px | py-1.5 등으로 작을 수 있음 | 보완 대상 |
| 선택 상한 | 보통 없음 또는 넉넉히 | 100자 제한 | 유지 (과다 입력만 제한) |

## 3. UX 픽스 계획 (적용/우선순위)

### 3.1 즉시 적용 (이미 반영 또는 코드 수준 픽스)
- **터치 타겟**: 칩 `min-height` 44px 근처, `py-2.5` 등 패딩 확대.
- **안내 문구**: “1개 이상 골라주세요” 또는 “원하는 만큼 선택해도 됩니다” 등으로 **최소 1개 선택** 인지 + 부담 완화.
- **placeholder**: BDSM 예시 유지 (예: `Sub · Switch · 감성 연애`).

### 3.2 단기 (다음 스프린트)
- **용어 설명(글로서리)**  
  - 옵션 A: 칩/라벨에 `title` 툴팁(한 줄 정의). 데스크톱 호버·모바일 포커스 시 기본 툴팁 노출.  
  - 옵션 B: “용어가 어렵다면?” → `/legal/glossary` 또는 모달에서 **역할·취향·관계** 용어만 짧게 정의.  
  - Feeld처럼 “길게 누르면 설명”은 터치 디바이스에서 구현 비용이 있으므로, 먼저 A 또는 정적 페이지로 제공 후 검토.

### 3.3 유지 (의도적 선택)
- **직접 입력 허용**: 서비스 규모·언어를 고려해 자유 입력 유지.
- **선택 개수 상한 없음(실질)**: 100자로 과다만 방지, “3~5개 권장” 등은 선택(과한 안내는 생략 가능).
- **저장 형식**: 기존 `role` 단일 문자열 + ` · ` 구분, 프로필 `splitTags` 표시 유지.
