# 욕망백서 알림 / 재방문 UX 문서 v0.1

## 1. 목적

- 욕망백서의 재방문 흐름을 설계한다.
- PWA 환경에서도 사용자가 자연스럽게 다시 들어오게 만드는 알림 및 리텐션 구조를 정리한다.

## 2. 전제

- 욕망백서는 PWA 우선 전략을 사용한다.
- 따라서 앱 푸시와 동일한 수준의 알림을 전제로 설계하지 않는다.
- 재방문은 `알림`, `상태 변화`, `콘텐츠 탐색 욕구` 세 축으로 설계한다.

## 3. 재방문의 핵심 원인

### 3-1. 상태 변화

- DM 요청이 도착함
- 보낸 요청이 수락됨
- 요청이 거절됨
- 요청이 만료되고 환불됨

### 3-2. 커뮤니티 흐름

- 새 글이 올라옴
- 관심 있는 분위기의 사람이 활동함
- 피드 밀도가 유지됨

### 3-3. 개인 행동 유도

- 내 자동 소개글 확인
- 내 프로필 보완
- 보유 포인트 사용

## 4. 알림 우선순위

### 우선순위 A

- 내가 받은 DM 요청
- 내가 보낸 요청의 수락
- 내가 보낸 요청의 거절
- 내가 보낸 요청의 만료 및 환불

### 우선순위 B

- 신고 처리 결과
- 포인트 충전 완료
- 수동 포인트 조정

### 우선순위 C

- 새로운 글 추천
- 최근 활동 안내

## 5. 알림 채널 설계

### 5-1. 인앱 알림

역할:

- 로그인 상태에서 가장 기본이 되는 채널

알림 항목 예:

- 새로운 대화 요청이 도착했습니다
- 요청이 수락되었습니다
- 45P가 환불되었습니다

### 5-2. 웹 푸시

역할:

- PWA 설치 사용자 대상 핵심 상태 변화 알림

우선 적용 대상:

- DM 요청 도착
- 요청 수락
- 거절/만료 환불

주의:

- 너무 많은 푸시는 거부감을 만든다
- 콘텐츠 추천 푸시는 후순위

### 5-3. 이메일

역할:

- 핵심 운영 알림 보조
- 비활성 사용자 재유입 보조

권장 사용처:

- 요청 수락 안내
- 포인트 충전 또는 환불 내역
- 정책 관련 중요한 변경

## 6. 알림 센터 정보 구조

```text
[상단]
알림

[필터]
전체 / DM / 포인트 / 운영

[리스트]
알림 카드 반복
```

### 카드 정보

- 제목
- 짧은 설명
- 시각
- 관련 화면 이동 CTA

예:

- 요청이 수락되었습니다
- 이제 자유롭게 대화를 시작할 수 있습니다
- 5분 전

## 7. 재방문 UX 시나리오

### 시나리오 1: 받은 DM 요청

유저가 느껴야 하는 것:

- 누군가가 나에게 관심을 보였다
- 지금 들어가 보면 될 것 같다

추천 알림 문구:

- 새로운 대화 요청이 도착했습니다
- 프로필을 보고 응답할지 결정해보세요

### 시나리오 2: 보낸 요청 수락

유저가 느껴야 하는 것:

- 지금 대화가 열렸다

추천 알림 문구:

- 요청이 수락되었습니다
- 이제 대화를 시작할 수 있습니다

### 시나리오 3: 거절 또는 만료

유저가 느껴야 하는 것:

- 손해가 확정된 게 아니라 결과가 정리됐다

추천 알림 문구:

- 요청이 거절되어 45P가 환불되었습니다
- 요청이 만료되어 90P가 전액 환불되었습니다

### 시나리오 4: 피드 재방문

유저가 느껴야 하는 것:

- 지금 들어가면 볼 만한 분위기가 있다

추천 문구:

- 최근 올라온 분위기를 둘러보세요
- 새로운 글이 이어지고 있습니다

## 8. 홈 재방문 장치

- 상단에 짧은 리듬 문장 노출
- 신규 글 필터 유지
- 최근 활동이 보이는 피드 밀도 유지
- 빈 화면보다 항상 무언가 읽을 수 있게 구성

## 9. 포인트 재방문 장치

- 여성 가입 보너스 270P를 가입 직후 분명하게 보여줌
- 남성은 충전 유도보다 `새 요청에만 포인트 사용` 메시지를 먼저 줌
- 환불 발생 시 즉시 체감할 수 있게 포인트 변화를 보여줌

## 10. 과한 리텐션 설계로 보일 수 있는 것

- 너무 잦은 푸시
- 감정 자극형 문구
- 핫한 상대가 기다린다는 식의 과장
- 홈에서 반복적인 충전 압박

## 11. 추천 마이크로카피

- 새로운 대화 요청이 도착했습니다
- 요청이 수락되었습니다
- 45P가 환불되었습니다
- 최근 올라온 분위기를 둘러보세요
- 내 프로필을 조금 더 채워보세요

## 12. 최종 판단

욕망백서의 재방문 UX는 중독 설계가 아니라  
`상태 변화와 사람의 흐름을 자연스럽게 다시 보게 만드는 설계`여야 한다.

PWA에서는 이 균형이 특히 중요하다.
