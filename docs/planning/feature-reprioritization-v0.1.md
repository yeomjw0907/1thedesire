# 욕망백서 기능 재정렬 및 우선순위 재분류 v0.1

## 1. 목적

- 현재까지 정리된 포지셔닝, 시장조사, 정책을 기준으로 기능 우선순위를 다시 잡는다.
- 무엇을 먼저 만들고 무엇을 뒤로 미룰지 명확히 한다.

## 2. 기능 재정렬의 이유

초기 문서는 `포인트 기반 DM`에 무게가 실려 있었다.  
하지만 현재 욕망백서의 방향은 아래에 가깝다.

- 피드 기반 취향 탐색
- 프로필 중심 판단
- 합의형 DM
- 운영 신뢰

따라서 기능 우선순위도 이 흐름에 맞게 바뀌어야 한다.

## 3. 새로운 우선순위 원칙

### 3-1. 탐색이 대화보다 먼저다

- 유저는 먼저 피드와 프로필로 상대를 판단해야 한다.
- DM은 탐색 이후의 행동이다.

### 3-2. 신뢰가 화려함보다 먼저다

- 포인트 규칙, 차단, 신고, 상태표시는 숨기면 안 된다.
- 운영 신뢰가 약하면 전체 구조가 무너진다.

### 3-3. 커뮤니티 밀도가 중요하다

- 첫 화면에서 사람이 보이고 분위기가 느껴져야 한다.
- 단순 매칭앱처럼 비면 안 된다.

## 4. Must Have

### 4-1. 가입 및 온보딩

- 간편 로그인
- 가입 폼
- 자기소개 입력
- 19세 이상 체크
- 여성 270P 지급
- 자동 소개글 생성

### 4-2. 피드

- 홈 피드 목록
- 글쓰기
- 이미지 1장 첨부
- 게시글 삭제
- 기본 필터 `전체 / 신규`

### 4-3. 프로필

- 닉네임, 연령대, 지역, 성향, 자기소개
- 최근 게시글 노출
- DM 요청 버튼
- 신고/차단 버튼

### 4-4. DM 및 채팅

- 90P 차감
- 요청 대기 상태
- 수락
- 거절
- 24시간 만료
- 거절 45P 환불
- 만료 90P 환불
- 차단 시 환불 없음
- 수락 후 무료 대화

### 4-5. 운영

- 신고 접수
- 차단
- 계정 제한
- 포인트 거래 이력

## 5. Should Have

### 5-1. 프로필 강화

- 성향 태그 구조화
- 프로필 사진
- 최근 활동 표시
- DM 받기 상태 표시

### 5-2. 피드 강화

- 태그 검색
- 지역 필터
- 성향 필터
- 자동 생성 게시글 템플릿 다양화

### 5-3. 운영 강화

- 신고 사유 세분화
- 게시글 숨김 자동화
- 운영 액션 로그

## 6. Could Have

- 프로필 공개 범위 제어
- 캡처 방지 또는 워터마크
- 성향 테스트
- 오프라인 만남 전 안전 기능
- 알림 고도화

## 7. 지금 빼는 것이 맞는 것

- 세이프 워드
- 무거운 인증 UX
- 추천 알고리즘
- 과한 등급제/게임화
- 복잡한 소셜 기능
- 음성/영상

## 8. 기능적으로 가장 달라진 부분

### 이전 관점

- 과금형 DM이 핵심

### 현재 관점

- 분위기 탐색형 커뮤니티가 핵심
- DM은 그 다음 단계

즉, 이제 진짜 MVP 핵심은 아래 순서다.

1. 피드 품질
2. 프로필 판단력
3. DM 상태 전이
4. 포인트 투명성
5. 신고/차단

## 9. 다음 구현 순서 권장

### 1단계

- 가입
- 프로필
- 자동 포스팅
- 홈 피드

### 2단계

- 프로필 상세
- 검색
- DM 요청
- 상태 기반 환불

### 3단계

- 채팅
- 포인트 내역
- 신고/차단
- 운영자 기본 도구

## 10. 최종 판단

욕망백서는 더 이상 `DM 기능이 있는 앱`이 아니라 `피드와 프로필을 통해 대화가 열리는 서비스`로 봐야 한다.

그래서 기능 판단 기준도 항상 아래 질문으로 돌아와야 한다.

- 이 기능이 탐색 경험을 좋게 만드는가
- 이 기능이 대화 전 불안을 줄이는가
- 이 기능이 서비스 신뢰를 높이는가

이 기준을 만족하지 못하면 우선순위를 낮추는 것이 맞다.
