# 욕망백서 UI 컴포넌트 시스템 v0.1

## 1. 목적

- 디자인 시스템과 카피 가이드를 실제 UI 단위로 변환한다.
- 개발과 디자인이 같은 컴포넌트 언어를 보도록 만든다.

## 2. 공통 원칙

- 컴포넌트는 정보 위계가 분명해야 한다
- 상태는 문구와 색을 함께 사용해 전달한다
- 과금, 환불, 차단 같은 민감 정보는 숨기지 않는다
- 텍스트가 중심이고 이미지는 보조다

## 3. App Shell

### 상단 바

구성:

- 로고 또는 화면명
- 우측 액션 1~2개
- 필요 시 포인트 잔액

원칙:

- 과도한 아이콘 나열 금지
- 현재 위치를 분명히 보여줄 것

### 하단 탭 바

탭:

- 홈
- 검색
- DM
- 포인트
- 프로필

원칙:

- 라벨은 반드시 함께 노출
- 활성 탭은 색상과 타이포 둘 다 바뀌어야 함

## 4. 버튼 시스템

### Primary Button

용도:

- 핵심 행동

예:

- 시작하기
- 요청 보내기
- 충전하기

스타일:

- `desire-500` 배경
- 밝은 텍스트
- 높이 48~52
- radius 14

### Secondary Button

용도:

- 대안 행동

예:

- 프로필 보기
- 수정하기
- 다시 보기

스타일:

- 어두운 surface 배경
- 얇은 border

### Danger Button

용도:

- 차단, 삭제, 제재 관련 액션

원칙:

- 무조건 빨갛게 크게 만들지 않는다
- 확인 단계가 필요한 액션에 사용

## 5. 카드 시스템

### Feed Card

구성:

- 작성자 메타
- 성향 태그 일부
- 본문
- 이미지 선택
- 하단 액션

원칙:

- 이미지보다 텍스트 우선
- 카드 높이는 너무 크지 않게 유지
- 프로필로 이동하고 싶게 만드는 정보 밀도 필요

### Profile Summary Card

구성:

- 닉네임
- 연령대 / 지역
- 성향
- 최근 활동 또는 상태

원칙:

- 시각적 자극보다 판단 재료를 먼저 준다

### Status Card

용도:

- 환불 완료, 요청 거절, 만료 안내

원칙:

- 색상 + 짧은 문구 + 보조 설명 조합
- 사용자가 현재 상태를 즉시 이해해야 함

## 6. 태그 / 배지 시스템

### 성향 태그

형태:

- pill 형태
- 톤다운된 배경색
- 작은 사이즈

원칙:

- 형광 느낌 금지
- 개수를 과하게 노출하지 않음

### 상태 배지

상태:

- 응답 대기중
- 수락됨
- 거절됨
- 만료됨
- 환불 완료

원칙:

- 짧은 문구 우선
- 색만으로 의미를 전달하지 않음

## 7. 입력 시스템

### Text Field

용도:

- 닉네임, 지역, 성향

원칙:

- 라벨 상시 노출
- placeholder는 설명 보조만

### Text Area

용도:

- 자기소개, 게시글 작성

원칙:

- 충분한 높이 확보
- 작성 힌트는 브랜드 톤 유지

예시 힌트:

- 대화를 시작하기 전에 보여주고 싶은 분위기를 적어보세요

## 8. DM 전용 컴포넌트

### DM Policy Panel

용도:

- 과금/환불 규칙 요약

구성:

- 90P 차감
- 거절 시 45P 환불
- 미응답 시 전액 환불
- 수락 후 무료 대화

원칙:

- 작은 표처럼 정리
- 겁주지 않고 이해시키는 구조

### DM Status Banner

용도:

- 채팅방 상태 표시

상태별 문구:

- 응답을 기다리는 중입니다
- 대화를 시작할 수 있습니다
- 요청이 거절되었습니다
- 요청이 만료되었습니다
- 더 이상 대화를 이어갈 수 없습니다

### Message Bubble

원칙:

- 버블 대비는 충분히 주되 과하게 장난스럽지 않게
- 시스템 메시지는 일반 메시지와 확실히 구분

## 9. 피드백 컴포넌트

### Toast

용도:

- 환불 완료
- 신고 접수
- 차단 완료

원칙:

- 짧고 명확한 문구
- 2초 내외 노출

### Empty State

용도:

- 피드 비어있음
- DM 없음
- 검색 결과 없음

원칙:

- 건조하지 않되 과장되지 않게
- 다음 행동을 한 줄로 제시

## 10. 모달 / 바텀시트

### Confirm Sheet

용도:

- DM 요청 확정
- 차단 확인
- 게시글 삭제 확인

원칙:

- 제목
- 핵심 설명
- 정책 요약
- 취소 / 확인

### Report Sheet

용도:

- 신고 사유 선택
- 상세 내용 입력

원칙:

- 지나치게 길지 않게
- 빠르게 신고할 수 있게

## 11. 우선 구현할 컴포넌트 세트

1. App Shell
2. Primary / Secondary / Danger Button
3. Feed Card
4. Profile Summary Card
5. DM Policy Panel
6. DM Status Banner
7. Message Bubble
8. Confirm Sheet
9. Toast
10. Tag / Badge

## 12. 최종 판단

욕망백서 컴포넌트 시스템의 핵심은 `섹시함`이 아니라 `정리된 긴장감`이다.

모든 컴포넌트는 아래를 만족해야 한다.

- 정보가 먼저 읽히는가
- 상태가 분명한가
- 브랜드 톤이 싸구려로 무너지지 않는가
- 20대 후반~30대 후반 사용자에게 신뢰를 주는가
