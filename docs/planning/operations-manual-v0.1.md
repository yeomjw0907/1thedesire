# 욕망백서 운영 매뉴얼 초안 v0.1

## 1. 목적

- 대표 1인 운영 기준으로 일상 운영 절차를 정리한다.
- 신고, 차단, 환불, 포인트, 계정 제재를 일관되게 처리하기 위한 기본 매뉴얼을 만든다.

## 2. 운영 원칙

- 사용자 보호가 우선이다
- 애매한 상황은 오래 끌기보다 제한 후 검토가 낫다
- 운영 판단은 짧고 명확해야 한다
- 모든 조치는 기록이 남아야 한다
- 감정 대응보다 상태 확인과 사실 확인이 먼저다

## 3. 일일 운영 루틴

### 오전 체크

- 신규 가입 수 확인
- 미처리 신고 확인
- 전날 DM 요청/거절/만료/환불 수 확인
- 차단 급증 여부 확인

### 오후 체크

- 신규 신고 처리
- 포인트/환불 이슈 확인
- 위험 계정 검토

### 마감 전 체크

- 미처리 신고 0에 가깝게 유지
- 환불 누락 여부 확인
- 반복 신고 계정 메모

## 4. 신고 처리 절차

### Step 1. 신고 접수 확인

볼 것:

- 신고 대상 유형
- 신고 사유
- 신고 원문
- 신고자와 대상 계정 이력

### Step 2. 위험도 판단

- Level 1: 일반 분쟁
- Level 2: 반복 요청, 외부 연락처 유도, 금전 요구
- Level 3: 미성년 의심, 협박, 불법 콘텐츠

### Step 3. 임시 조치

- 위험도가 높으면 우선 숨김 또는 제한
- 게시글은 숨김
- 계정은 `restricted` 또는 `suspended`

### Step 4. 최종 처리

- `warn`
- `restrict`
- `suspend`
- `ban`
- `dismissed`

### Step 5. 로그 남기기

- 대상
- 조치
- 사유
- 시각

## 5. 차단 관련 운영 기준

### 사용자 차단

- 즉시 반영이 원칙
- 별도 승인 불필요

### 운영 확인 포인트

- 특정 계정이 비정상적으로 많이 차단당하는지
- 차단과 신고가 같이 늘어나는지

### 조치 기준

- 차단 누적이 높은 계정은 우선 검토
- 반복되면 `restrict` 또는 `suspend`

## 6. 포인트 / 환불 운영 절차

### 기본 정책

- 새 DM 요청 시 90P 차감
- 거절 시 45P 환불
- 24시간 미응답 시 90P 환불
- 차단 시 환불 없음

### 운영자가 확인할 것

- 환불이 상태에 맞게 반영됐는지
- 중복 환불이 없는지
- 수동 조정이 필요한지

### 수동 조정이 필요한 예외

- 결제 오류
- 중복 반영
- 운영 실수

## 7. 계정 제재 운영 기준

### warn

- 경미한 규칙 위반
- 저급한 표현
- 1회성 무례함

### restrict

- 반복 스팸
- 반복적 원치 않는 요청
- 서비스 톤 훼손

### suspend

- 외부 연락처 유도
- 금전 요구
- 강한 괴롭힘
- 미성년 의심 검토 중

### ban

- 협박
- 불법 촬영물 / 유출물
- 사칭
- 명백한 사기
- 미성년 관련 위반

## 8. 자주 발생할 수 있는 운영 상황

### 상황 1. 사용자가 "왜 포인트가 차감됐나요?"라고 묻는 경우

답변 기준:

- 새로운 DM 요청 시 90P가 차감됩니다
- 상태에 따라 환불 규칙이 다릅니다
- 현재 요청 상태와 거래 내역을 함께 안내합니다

### 상황 2. 사용자가 "왜 환불이 안 되나요?"라고 묻는 경우

답변 기준:

- 거절과 미응답 만료는 환불 대상입니다
- 차단된 요청은 환불되지 않습니다
- 실제 상태를 먼저 확인한 뒤 안내합니다

### 상황 3. 사용자가 "이 사용자가 불편해요"라고 말하는 경우

답변 기준:

- 차단과 신고를 우선 안내
- 필요 시 운영 검토로 연결

## 9. 운영 응답 톤

- 짧고 명확하게
- 감정적으로 맞받아치지 않기
- 법률 해석처럼 말하지 않기
- 현재 상태와 정책 기준을 분리해서 설명하기

예:

- 현재 요청은 만료 상태이며, 전액 환불이 반영되었습니다
- 차단된 요청은 환불 대상이 아닙니다
- 신고가 접수되어 검토 중입니다

## 10. 운영자 체크리스트

- 신고를 읽고 바로 위험도를 분류했는가
- 필요한 임시 조치를 먼저 했는가
- 최종 조치를 기록했는가
- 포인트/환불 상태를 함께 확인했는가
- 반복 계정 메모를 남겼는가

## 11. 최종 판단

욕망백서 운영 매뉴얼의 핵심은 `빠른 보호`, `일관된 처리`, `기록 유지`다.

화려한 운영 체계보다, 같은 상황에 같은 대응을 할 수 있는 기준이 더 중요하다.
