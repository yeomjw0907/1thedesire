# 욕망백서 운영자 / 관리자 화면 명세 v0.1

## 1. 목적

- 대표 1인 운영 기준으로 필요한 관리자 화면 구조를 정의한다.
- 신고, 제재, 포인트, DM 상태를 최소한의 부담으로 관리할 수 있게 한다.

## 2. 운영자 콘솔의 역할

운영자 화면은 백오피스가 아니라 `서비스 신뢰를 유지하는 제어판`이다.

핵심 목표:

- 위험 계정 빠른 식별
- 신고 처리 속도 확보
- 포인트 분쟁 대응
- DM 상태 및 환불 흐름 확인

## 3. 운영 우선순위

1. 미성년 의심
2. 협박, 금전 요구, 외부 연락처 유도
3. 불법 촬영물 또는 유포 시도
4. 반복 스팸 요청
5. 일반 분쟁

## 4. 관리자 화면 IA

```text
대시보드
유저 관리
신고 관리
게시글 관리
DM / 요청 상태 관리
포인트 / 결제 관리
운영 로그
정책 메모
```

## 5. 대시보드

### 목적

- 운영자가 오늘 처리해야 할 리스크를 먼저 본다

### 핵심 위젯

- 신규 신고 수
- 미처리 신고 수
- 오늘 생성된 DM 요청 수
- 만료된 요청 수
- 환불 발생 수
- 차단 발생 수
- 신규 가입 수

### 상단 경고 영역

- 미성년 의심 계정
- 반복 신고 계정
- 반복 차단 유발 계정

## 6. 유저 관리

### 목적

- 특정 유저의 상태를 빠르게 확인하고 조치한다

### 리스트 컬럼

- 닉네임
- 성별
- 연령대
- 지역
- 가입일
- 계정 상태
- 신고 누적
- 차단 누적
- 현재 포인트

### 상세 화면

- 프로필 정보
- 최근 게시글
- 최근 DM 요청 이력
- 신고 이력
- 포인트 거래 이력
- 제재 이력

### 운영 액션

- 경고
- 제한
- 일시정지
- 영구정지
- 수동 포인트 조정

## 7. 신고 관리

### 목적

- 신고를 유형별로 빠르게 분류하고 처리한다

### 리스트 컬럼

- 신고 접수 시각
- 신고자
- 대상
- 신고 유형
- 상태
- 우선순위

### 필터

- 전체
- 미처리
- 검토중
- 해결됨
- 기각

### 상세 화면

- 신고 대상 유형
- 원문 또는 대상 링크
- 신고 사유
- 신고 상세 설명
- 관련 계정 이력

### 운영 액션

- 경고 처리
- 게시글 숨김
- 계정 제한
- 영구정지
- 기각

## 8. 게시글 관리

### 목적

- 문제 게시글을 빠르게 숨기거나 확인한다

### 리스트 컬럼

- 작성자
- 작성 시각
- 자동 생성 여부
- 상태
- 신고 수

### 액션

- 숨김
- 복구
- 삭제
- 작성자 프로필 보기

## 9. DM / 요청 상태 관리

### 목적

- 분쟁과 환불이 얽힌 DM 상태를 추적한다

### 리스트 컬럼

- 요청 시각
- 발신자
- 수신자
- 현재 상태
- request cost
- refund amount
- expires at

### 필터

- pending
- agreed
- declined
- expired
- blocked

### 상세 화면

- 상태 전이 로그
- consent event 로그
- 환불 처리 로그
- 차단 여부
- 신고 여부

## 10. 포인트 / 결제 관리

### 목적

- 포인트 분쟁 대응과 충전 확인

### 리스트 컬럼

- 거래 시각
- 유저
- 유형
- 금액
- balance after
- reference type
- policy code

### 결제 이벤트 화면

- 결제 수단
- 상태
- 금액
- raw payload 요약

### 운영 액션

- 수동 지급
- 수동 차감
- 거래 메모 추가

## 11. 운영 로그

### 목적

- 누가 어떤 제재를 했는지 기록한다

### 포함 내용

- 대상 유저
- 액션 타입
- 사유
- 시각
- 메타데이터

## 12. 정책 메모 화면

### 목적

- 현재 운영 기준을 운영자가 빠르게 확인한다

### 포함 내용

- 신고 우선순위
- 환불 기준
- 차단 시 정책
- 미성년 의심 대응 기준
- 외부 연락처/금전 요구 대응 기준

## 13. 관리자 UX 원칙

- 아름답기보다 빠르고 명확해야 한다
- 상태는 색상 + 텍스트 조합으로 보여준다
- 위험도 높은 항목이 먼저 보여야 한다
- 한 화면에서 핵심 조치까지 갈 수 있어야 한다

## 14. 관리자 카피 원칙

- 지나치게 친절할 필요는 없다
- 짧고 명확하게
- 법률 판단처럼 보이기보다 운영 판단 문구로 유지

예:

- 미처리 신고가 있습니다
- 이 요청은 환불이 완료되었습니다
- 반복 차단 이력이 확인됩니다

## 15. 최종 판단

욕망백서 관리자 화면은 화려한 백오피스가 아니라  
`빠르게 위험을 발견하고, 조용히 안정화하는 도구`여야 한다.

대표 1인이 보기 때문에 무엇보다 `읽기 쉬움`과 `상태 명확성`이 중요하다.
