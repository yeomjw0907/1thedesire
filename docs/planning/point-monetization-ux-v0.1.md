# 욕망백서 포인트 / 과금 UX 상세 문서 v0.1

## 1. 목적

- 포인트 시스템을 단순 결제가 아니라 사용자 경험 흐름으로 정리한다.
- 과금 불신을 줄이고, 대화 전환을 방해하지 않는 구조를 만든다.

## 2. 핵심 원칙

- 과금은 `새로운 요청`에서만 발생한다
- 연결 이후 대화는 무료다
- 환불 규칙은 미리 알려준다
- 여성 유저 메리트는 명확하게 드러낸다
- 포인트는 기능보다 `행동 설계 장치`다

## 3. 현재 과금 구조

### 공통

- 새로운 DM 요청 시 `90P` 차감

### 여성

- 가입 시 `270P` 지급
- 받은 DM 수락 및 대화 무료
- 새로운 DM 발신 시에는 공통 규칙 적용

### 남성

- 새로운 DM 발신 시 `90P` 차감
- 받은 DM 수락 및 연결 후 대화는 무료

### 환불

- 거절 시 `45P` 환불
- 24시간 미응답 만료 시 `90P` 전액 환불
- 차단 시 환불 없음

## 4. 사용자 심리 기준으로 본 과금 UX

### 사용자가 가장 싫어하는 것

- 왜 돈이 나갔는지 모르는 상태
- 응답이 없는데 돈만 사라지는 경험
- 이미 열린 대화에서 또 결제하는 경험
- 숨겨진 규칙이 많은 구조

### 사용자가 받아들이는 것

- 먼저 말을 거는 비용
- 상태에 따라 일부 또는 전체 환불되는 구조
- 수락 후 무료 대화
- 여성에게 더 낮은 진입장벽을 주는 구조

## 5. 포인트 UX 흐름

### 흐름 1. 보유 포인트 확인

- 프로필 또는 상단에서 현재 잔액 확인
- 부족할 경우 즉시 충전으로 이동

### 흐름 2. DM 요청 전

- 90P 차감 안내
- 거절/만료 환불 규칙 안내
- 수락 후 무료 대화 안내

### 흐름 3. DM 요청 후

- 상태별 결과를 명확히 보여줌
- 거절/만료 시 환불이 즉시 반영되었음을 안내

### 흐름 4. 포인트 내역 확인

- 차감
- 환불
- 충전
- 가입 보너스

## 6. 핵심 UX 화면별 요구사항

### 프로필 화면

- DM 버튼 근처에 핵심 정책 2줄 노출

예:

- 요청 시 90P 차감
- 거절 시 45P 환불 · 미응답 시 전액 환불

### DM 요청 팝업

- 가장 중요한 과금 설명 화면
- 숫자는 크게, 문장은 짧게

### DM 목록

- 요청 상태와 환불 여부를 바로 보여줘야 함

예:

- 거절됨 / 45P 환불 완료
- 만료됨 / 90P 환불 완료

### 포인트 화면

- 현재 잔액
- 거래 내역
- 환불 내역
- 여성 혜택 안내

## 7. 포인트 화면 정보 구조

```text
[잔액 카드]
현재 보유 포인트

[설명 카드]
새로운 대화 요청에만 포인트가 사용됩니다
수락된 대화는 추가 과금 없이 이어집니다

[거래 내역 탭]
전체 / 차감 / 환불 / 충전
```

## 8. 카피 원칙

- 결제보다 요청 구조를 먼저 설명한다
- 환불 규칙을 숨기지 않는다
- 사용자가 손해보는 느낌을 줄이는 문구를 쓴다

좋은 예:

- 새로운 요청에만 포인트가 사용됩니다
- 수락 후 대화는 무료입니다
- 응답이 없으면 전액 환불됩니다

나쁜 예:

- 지금 결제하고 바로 시작
- 빠르게 충전하고 연결
- 늦기 전에 결제

## 9. 여성 메리트 노출 원칙

- 과장하지 않고 분명하게 보여준다
- 서비스 전반에 최소 3곳 이상 노출 가능

추천 노출 위치:

1. 가입 완료 화면
2. 프로필 화면 보조 문구
3. 포인트 화면 설명 카드

## 10. 과금 UX에서 피해야 할 것

- 홈에서 과도한 충전 유도
- 모든 행동에 포인트를 붙이는 구조
- 연결 후 추가 차감
- 환불 조건 숨김
- 남성 과금만 강조하고 여성 메리트를 과장되게 광고하는 카피

## 11. 향후 확장 가능성

- 첫 충전 보너스
- 특정 기간 프로모션 포인트
- DM 묶음 패키지

단, 아래는 주의:

- 랜덤형 보상
- 도박형 포인트 이벤트
- 과도한 미끼 프로모션

## 12. 최종 판단

욕망백서의 포인트 UX는 `돈을 쓰게 만드는 구조`보다  
`먼저 말을 걸어도 불안하지 않은 구조`에 가까워야 한다.

그래야 결제 전환도 유지되고, 서비스 톤도 무너지지 않는다.
