# 홈 중심 페이지 상세 구조안 v0.1

## 1. 홈

### 목적

- 게시글 피드로 탐색과 전환을 동시에 처리한다.

### 상단

- 로고 또는 서비스명
- 글쓰기 버튼
- 새로고침 버튼
- 필터 열기 버튼

### 필터/정렬

- 기본 정렬
- `최신순`
- `인기순`

- 기본 필터
- `성향`
- `성별`
- `최근검사 확인`
- `인증된 사용자`

- 확장 필터
- `지역`
- `조회수`
- `사진 포함`

### 게시글 카드

- 프로필 이미지
- 닉네임
- 기본 정보: 나이대, 지역, 성향
- 글 내용
- 첨부 이미지
- 메타 정보: 작성 시각, 조회수 등
- CTA: `DM 보내기`

### 인터랙션

- 프로필 이미지 클릭 -> 프로필 상세
- 닉네임 클릭 -> 프로필 상세
- DM 버튼 클릭 -> DM 요청 시트 또는 DM 작성 진입

### 비고

- `프로필 보기` 텍스트 버튼은 두지 않는다.
- 카드가 과하게 복잡해지지 않도록 DM 버튼은 보조 CTA로 둔다.

## 2. DM

### 목적

- 사용자의 관점에서 DM을 더 쉽게 관리한다.

### 탭 구조

1. `전체`
- 내가 보낸 DM
- 나에게 온 요청
- 진행 중인 대화 포함

2. `읽지 않음`
- 상대가 아직 열지 않았거나
- 답변이 없는 상태의 건

3. `요청`
- 다른 사용자가 나에게 보낸 신규 요청

### 상태 원칙

- `거절` 기능은 제거
- `읽음/미읽음`과 `대화 진행 여부`를 분리해서 본다
- 장시간 무응답은 `만료` 또는 별도 종료 정책으로 처리

### 추가 권장

- `요청` 탭에 숫자 배지 표시
- 상대 조회 여부가 가능하면 `읽음`으로 상태 반영

## 3. 포인트

### 목적

- 충전과 안내를 실전형 결제 UX로 정리한다.

### 계좌 정보

- 은행: `우리은행`
- 계좌번호: `1005-90-369058`
- 예금주: `주식회사 98점7도`

### 기본 구조

- 현재 보유 포인트
- 충전 상품 선택
- 입금 금액 확인
- 입금자명 입력
- 오류 시 환불 계좌 입력
- 결제 진행 버튼

### 핵심 안내

- `결제 후 1~3분 내 자동 충전됩니다`
- `입금자명을 정확히 입력해야 자동 충전됩니다`
- `오입금/충전 오류 시 이메일 문의`
- `중요 대화 내용은 별도 보관 권장`

### 긴 안내 문구 처리 원칙

- 화면에 전부 노출하지 않는다
- `안내사항 더보기` 또는 `충전/환불 정책 보기`로 분리한다
- 법률성 문구는 별도 정책 문서와 연결한다

### 반영할 참고 포인트

- 소액 충전 권장
- 입금자명 제한 규칙
- 충전 오류 시 새로고침 유도
- 문의 메일 명시
- 포인트 사용처 안내

## 4. 프로필

### 목적

- 사용자가 DM 전송 전 최종 판단을 내리는 페이지로 강화한다.

### 상단

- 프로필 이미지
- 닉네임
- 기본 정보: 나이대, 지역, 성별
- 대표 성향

### 신뢰/성향 섹션

- 상세 성향 태그
- 최근검사 확인 배지
- 신뢰 정보
- 최근 활동 정보

### 본문

- 자기소개
- 최근 게시글
- 첨부 이미지가 있는 최근 글 우선 노출 가능

### 하단 CTA

- `DM 보내기`

### 비고

- 프로필 이미지는 필수에 가깝게 검토
- 최근검사 확인은 프로필 상단 신뢰 섹션에 명확히 위치

## 5. 검색

### 방향

- 독립 페이지는 제거
- 홈 안으로 기능 흡수

### 홈으로 이동하는 기능

- 키워드 탐색
- 성향 필터
- 성별 필터
- 지역 필터
- 인증/최근검사 확인 필터

## 6. 구현 우선순위

1. 홈 개편
2. DM 탭 개편
3. 프로필 강화
4. 포인트 화면 개편
5. 검색 제거
