# 욕망백서 디자인 시스템 v1

## 1. 목적

- 욕망백서의 브랜드를 시각적으로 재정립한다.
- 20대 후반에서 30대 후반 중심의 국내 사용자에게 맞는 UI/UX 기준을 만든다.
- 저가형 성인앱, 야설사이트, 과도한 킹크앱처럼 보이지 않는 방향을 고정한다.

## 2. 브랜드 재정의

### 브랜드 핵심 문장

욕망백서는 욕망을 숨기게 하지 않되, 조잡하게 소비시키지도 않는 한국형 익명 취향 플랫폼이다.

### 시각적으로 전달해야 하는 감정

- 은밀함
- 정제됨
- 신뢰
- 밀도 있는 취향
- 너무 차갑지 않은 거리감

### 피해야 하는 감정

- 싸구려 유혹
- 퇴폐 업소 느낌
- 게임형 과금앱 분위기
- 과한 기술적 차가움
- 십대 감성이나 과한 트렌디함

## 3. 타겟 연령층에 맞는 디자인 판단

### 주요 타겟

- 27세에서 39세 전후
- 모바일 사용에 익숙하지만 지나치게 복잡한 UI는 싫어함
- 감각은 보되, 믿을 수 있어야 결제함
- `예쁜 앱`보다 `기분이 안전한 앱`을 선호함

### 이 연령층에 맞는 UX 특징

- 정보 구조가 명확해야 한다
- 버튼과 상태 변화가 즉시 이해되어야 한다
- 카피가 유치하거나 과장되면 바로 이탈한다
- 노골적인 성적 시각 요소보다 분위기와 품질을 더 본다

## 4. 시각 방향

### 콘셉트 이름

`Velvet Ledger`

의미:

- `Velvet`: 부드럽고 은밀한 촉감
- `Ledger`: 기록, 질서, 신뢰

이 콘셉트는 욕망과 질서를 동시에 잡기 위한 시각 방향이다.

## 5. 컬러 시스템

### 역할 정의

- 배경은 `깊고 밀도 있는 어둠`
- 액션 컬러는 `절제된 진홍`
- 신뢰/환불/안전은 `차가운 민트가 아니라 성숙한 청록`
- 본문 텍스트는 새하얗지 않은 `부드러운 아이보리 그레이`

## 6. 컬러 사용 원칙

- 검정 배경 위에 새빨간 버튼만 남발하지 않는다.
- 진홍은 `행동`과 `긴장감`에만 쓰고, 전체 화면을 붉게 물들이지 않는다.
- 청록은 안전, 환불, 상태 메시지에만 제한적으로 사용한다.
- 회색은 싸구려가 아니라 `가죽과 돌의 중간 질감`처럼 보여야 한다.

## 7. 타이포그래피

### 폰트 전략

- 국문: `Pretendard`
- 영문 및 숫자 포인트: `Instrument Serif` 또는 `Cormorant Garamond`
- UI 숫자/가격/시간: `Montserrat`

### 왜 이렇게 가는가

- Pretendard는 정보 전달력이 강하다.
- 영문 포인트 세리프를 일부 섞으면 브랜드 무드가 급격히 고급스러워진다.
- 가격과 숫자는 Montserrat로 정리하면 신뢰감이 생긴다.

### 타입 스케일

- `Display`: 34 / 40 / 600
- `H1`: 28 / 34 / 700
- `H2`: 22 / 28 / 700
- `H3`: 18 / 24 / 600
- `Body`: 15 / 24 / 400
- `Body-Strong`: 15 / 24 / 600
- `Caption`: 12 / 18 / 500
- `Micro`: 11 / 16 / 500

### 타이포 원칙

- 본문은 촘촘하지 않게 충분한 줄간격을 준다.
- 제목은 과장보다 밀도를 준다.
- 세리프는 포인트만 사용하고, UI 전체를 세리프로 덮지 않는다.

## 8. 공간 시스템

### 8pt 기반 스케일

- `4`, `8`, `12`, `16`, `20`, `24`, `32`

### 레이아웃 원칙

- 바깥 여백은 충분히 준다
- 카드 내부 여백은 16 이상
- 섹션 간 간격은 24 이상
- 작은 정보 조각은 촘촘하게, 큰 블록은 넓게

## 9. 형태 언어

### 모서리

- 기본 카드 radius: `20`
- 작은 칩 radius: `999`
- 버튼 radius: `14`
- 시트/팝업 radius: `24`

### 그림자와 테두리

- 무거운 그림자보다 얇은 테두리와 레이어 차이로 구분
- border 색은 `rgba(255,255,255,0.06)` 전후
- 유리 느낌보다 `밀도 있는 매트 질감`이 맞다

## 10. 컴포넌트 원칙

### 버튼

- Primary: 진홍 배경, 강한 행동 유도
- Secondary: 어두운 표면 + 밝은 텍스트
- Tertiary: 텍스트형

버튼 문구 원칙:

- `대화 요청 보내기`
- `프로필 보기`
- `신고하기`

짧고 명확해야 하며, 싸구려 유혹 문구를 쓰지 않는다.

### 카드

- 피드 카드와 프로필 카드 모두 `텍스트 밀도`가 중요하다
- 이미지는 보조다
- 카드 안에서 정보 우선순위가 분명해야 한다

### 배지 / 태그

- 성향 태그는 과한 형광 칩이 아니라 톤다운된 칩으로 처리
- 상태 배지는 색보다 문구를 우선

예:

- `응답 대기중`
- `거절됨`
- `환불 완료`

### 토스트 메시지

앱 전역에서 사용하는 피드백 알림. sonner 기반으로 통일.

위치: 상단 중앙 (`top-center`)

| 속성 | 값 |
|---|---|
| 배경 | `surface-750` (`#1D1D22`) |
| 텍스트 | `text-primary` (`#DDD6CF`) |
| 테두리 | `border-soft` (`rgba(255,255,255,0.06)`) |
| 모서리 | `radius-button` (`14px`) |
| 패딩 | `14px 18px` |
| 그림자 | `0 8px 24px rgba(0,0,0,0.4)` |
| 폰트 크기 | `14px` |
| 간격 (gap) | `8px` |

상태별 아이콘 색상:

- `error`: `danger` (`#C44444`)
- `success`: `success` (`#2F9A72`)
- `info`: `info` (`#4E7BB7`)

사용 규칙:

- 사용자 액션의 성공/실패 피드백에 사용
- 인라인 에러 대신 토스트로 통일 (폼 밸리데이션 포함)
- 문구는 짧고 명확하게, 과장 없이
- 자동 닫힘 (기본 4초)

예시:

- `toast.error('이메일 또는 비밀번호가 맞지 않습니다.')`
- `toast.success('인증 메일을 다시 보냈습니다.')`
- `toast.error('잠시 후 다시 시도해 주세요.')`

### 상태 배너

- DM 상태를 설명하는 상단 배너는 매우 중요
- 배경 톤 차이 + 작은 아이콘 + 짧은 설명 조합으로 설계

## 11. 화면별 무드 가이드

### 홈

- 가장 살아 있어야 하는 화면
- 어두운 바탕 위에 카드가 층층이 쌓이는 구조
- 과하게 차갑지 않게 텍스트 밀도를 살린다

### 프로필

- `사람 소개`보다 `취향 보고서`
- 질서 있고 정리된 화면
- 정보의 신뢰감이 가장 중요

### DM

- 돈을 내는 느낌보다 `요청 절차` 느낌이 나야 한다
- 환불 규칙은 긴장 완화 요소로 작동해야 한다

### 채팅

- 집중도가 높아야 한다
- 버블은 가볍고, 상단 상태는 분명해야 한다

## 12. 카피 스타일

### 해야 하는 말투

- 차분함
- 명확함
- 불필요한 과장 없음
- 친절하지만 과잉 친절하지 않음

### 하지 말아야 할 말투

- `지금 바로 야한 대화 시작`
- `핫한 사람들과 즉시 만남`
- `외로운 밤 해결`
- `자극적인 유혹 문구`

### 추천 카피 예시

- 취향은 숨기지 말고, 안전하게 나누세요
- 분위기를 먼저 보고, 대화는 그 다음에 시작하세요
- 요청은 신중하게, 대화는 명확하게

## 13. 아이콘과 이미지 원칙

- 아이콘은 얇고 절제된 라인 스타일
- 하트, 불꽃, 입술 같은 클리셰 아이콘 사용 금지
- 이미지는 얼굴 노출보다 무드 중심 비주얼도 허용
- 스톡 이미지 느낌이 강하면 피한다

## 14. 모션 원칙

- 모션은 의미 있을 때만 사용
- 페이지 진입 시 120~220ms 정도의 짧은 페이드와 슬라이드
- 카드 등장에 과한 탄성 금지
- DM 상태 변화는 작은 배너 전환으로 명확히 알림

## 15. 접근성과 사용성

- 텍스트 대비는 충분히 확보
- 작은 글씨는 `11px` 이하로 내리지 않는다
- 포인트, 환불, 경고, 차단 문구는 색만으로 전달하지 않는다
- 터치 타겟은 최소 `44px`

## 16. 경쟁 서비스 대비 디자인 차별화

### BLUV에서 가져올 것

- 신뢰와 프로필 중심 구조

### PURE에서 가져올 것

- 세련된 익명성

### Feeld에서 가져올 것

- 현대적이고 절제된 브랜드 언어

- 고급 편집지 같은 밀도
- 성숙한 사용자층에 맞는 차분함

## 18. 최종 판단

욕망백서 디자인의 핵심은 `야함`이 아니라 `품위 있는 긴장감`이다.

유저는 여기서 자극만 원하지 않는다.  
신뢰할 수 있는 분위기, 나와 비슷한 결의 사람들, 조심스럽지만 솔직한 공간을 원한다.

그래서 욕망백서 디자인은 항상 아래 질문을 통과해야 한다.

- 이 화면이 싸구려처럼 보이지 않는가
- 이 화면이 성숙한 사용자층에게 신뢰를 주는가
- 이 화면이 욕망을 숨기지 않되 조잡하게 소비하지 않는가
