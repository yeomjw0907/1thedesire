# 욕망백서 MVP 출시 체크리스트 v0.1

## 1. 목적

- MVP 출시 전에 반드시 확인해야 할 항목을 한 장으로 정리한다.
- 기획, 운영, 정책, UX, 데이터, 리스크 관점에서 누락을 줄인다.

## 2. 제품 핵심 흐름 점검

### 가입

- 간편 로그인이 정상 동작하는가
- 닉네임, 성별, 연령대, 지역, 성향, 자기소개 입력이 가능한가
- `19세 이상` 체크와 약관 동의가 필수로 작동하는가
- 가입 완료 후 자동 소개글이 생성되는가
- 여성 가입 시 270P 지급 흐름이 설계상 정리되어 있는가

### 홈 피드

- 홈 피드가 빈 화면처럼 느껴지지 않는가
- 자동 소개글 포함 피드 밀도가 유지되는가
- 글쓰기, 삭제, 프로필 이동 흐름이 자연스러운가

### 프로필

- 닉네임, 연령대, 지역, 성향, 자기소개, 최근 글이 판단에 충분한가
- 프로필에서 DM 요청 전에 필요한 정보가 보이는가
- 신고/차단이 찾기 쉬운 위치에 있는가

### DM

- 새 DM 요청 시 90P 차감 구조가 명확한가
- 수락 전 입력창이 비활성 처리되는가
- 거절 시 45P 환불 흐름이 보이는가
- 24시간 만료 시 전액 환불 흐름이 보이는가
- 수락 후 대화는 무료라는 점이 충분히 전달되는가

### 포인트

- 현재 포인트 잔액이 명확하게 보이는가
- 차감, 환불, 충전 내역이 구분되어 보이는가
- 과금 규칙이 숨겨지지 않는가

## 3. 운영 / 정책 점검

### 신고 / 차단

- 프로필, 게시글, 채팅에서 신고가 가능한가
- 차단 시 상호 노출과 DM이 제한되는가
- 신고 접수 후 운영 흐름이 정의되어 있는가

### 제재

- `warn`, `restrict`, `suspend`, `ban` 기준이 문서화되어 있는가
- 미성년 의심, 협박, 금전 요구, 불법 콘텐츠 시나리오가 정리되어 있는가

### 정책 노출

- DM 과금/환불 규칙이 프로필, 팝업, DM 상태 화면에 노출되는가
- 가입 시 자동 소개글 안내가 보이는가
- 차단 정책과 환불 없음이 사용자 행동 직전에 보이는가

## 4. 디자인 / UX 점검

### 브랜드 일관성

- 전체 화면이 저가형 성인앱처럼 보이지 않는가
- 자극보다 신뢰가 먼저 느껴지는가
- 20대 후반~30대 후반 사용자에게 유치하지 않은가

### 정보 구조

- 홈은 탐색, 프로필은 판단, DM은 요청 절차로 분리되어 보이는가
- 버튼과 상태 문구가 과도하게 많지 않은가
- 텍스트가 충분히 읽히는가

## 5. 알림 / 재방문 점검

- 요청 수락, 거절, 만료, 환불 알림 우선순위가 정의되어 있는가
- 첫 7일 리텐션 시나리오가 문서화되어 있는가
- PWA 기준 재방문 설계가 정리되어 있는가

## 6. 데이터 / 개발 문서 점검

- 상태도 기반 기능명세가 최신 기획과 맞는가
- DB 설계와 SQL 초안이 현재 정책과 일치하는가
- 마이그레이션 파일 설계 문서가 정리되어 있는가
- 포인트 거래와 DM 상태 전이 로그가 남는 구조인가

## 7. 법률 / 약관 문서 점검

- 이용약관 초안이 있는가
- 포인트 및 환불 정책이 있는가
- 운영정책이 있는가
- FAQ와 도움말이 준비되어 있는가

## 8. 출시 전 리스크 체크

- 미응답/거절/차단 상황의 사용자 불만 포인트가 정리되어 있는가
- 자동 소개글에 대한 사용자 반응을 모니터링할 준비가 되어 있는가
- 여성 메리트 메시지가 과장 광고처럼 보이지 않는가
- 지역 노출이 너무 세밀하지 않은가
- 신고/차단이 실제로 즉시 작동하는가

## 9. 출시 직후 반드시 볼 지표

- 가입 완료율
- 자동 소개글 삭제율
- Day 1 재방문율
- 첫 7일 프로필 보완율
- 첫 DM 요청률
- DM 수락률
- 거절률
- 만료율
- 환불 발생률
- 신고율
- 차단율

## 10. 출시 보류 신호

아래 중 하나라도 심하면 출시를 미루는 것이 맞다.

- 과금 규칙이 UI에서 충분히 설명되지 않음
- 신고/차단 흐름이 문서만 있고 실제 제품 흐름이 약함
- 자동 소개글 정책에 대한 내부 정리가 부족함
- DM 상태 전이와 환불 흐름이 일관되지 않음

## 11. 최종 판단

욕망백서 MVP의 출시는 기능 수가 아니라  
`신뢰가 깨지지 않는 기본 구조가 갖춰졌는가`로 판단해야 한다.

특히 DM, 환불, 신고, 차단 이 네 가지가 흔들리면 출시 후 바로 문제가 커진다.
