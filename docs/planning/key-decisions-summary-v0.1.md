# 욕망백서 핵심 의사결정 요약 v0.1

## 1. 목적

- 지금까지 합의된 핵심 결정을 한 장으로 요약한다.
- 새로 합류한 사람이 빠르게 현재 방향을 이해할 수 있게 한다.

## 2. 제품 방향

- 욕망백서는 `피드 기반 취향 커뮤니티 + 합의 기반 DM` 구조로 간다.
- 단순 성인 채팅앱이 아니라 `한국형 취향 안전 소셜`로 포지셔닝한다.
- 탐색이 대화보다 먼저다.

## 3. 출시 플랫폼

- 초기 출시는 `모바일 웹 / PWA`
- 앱스토어 정식 앱은 현재 기준 아님
- 추후 필요 시 `라이트 앱` 또는 `순화된 앱 버전` 별도 검토

## 4. 핵심 사용자 흐름

1. 가입
2. 자동 소개글 생성
3. 홈 피드 탐색
4. 프로필 판단
5. DM 요청
6. 수락/거절/만료
7. 수락 후 무료 대화

## 5. 가입 정책

- `19세 이상` 체크 필수
- 닉네임, 성별, 연령대, 지역, 성향, 자기소개 입력 필수
- 가입 완료 시 기본 소개글 자동 생성

## 6. 포인트 정책

### 공통

- 새 DM 요청 시 `90P` 차감

### 여성

- 가입 시 `270P` 지급
- 받은 요청 수락 및 대화 무료
- 새 DM 발신 시 90P 차감

### 남성

- 새 DM 발신 시 90P 차감
- 받은 요청 수락 후 대화 무료

## 7. 환불 정책

- 거절 시 `45P` 환불
- 24시간 미응답 만료 시 `90P` 전액 환불
- 차단 시 환불 없음

## 8. DM 상태 모델

- `pending`
- `agreed`
- `declined`
- `expired`
- `blocked`
- `closed`

## 9. 프로필 원칙

- 프로필은 `예쁜 카드`가 아니라 `판단 도구`
- 사진보다 취향, 자기소개, 최근 글이 먼저
- 지역은 광역 단위 중심

## 10. UX 원칙

- 자극보다 신뢰
- 익명성보다 통제감
- 홈은 분위기 탐색
- 프로필은 판단
- DM은 결제가 아니라 요청 절차

## 11. 브랜드 원칙

- 저가형 성인앱처럼 보이면 안 된다
- 세련되고 절제된 톤 유지
- 메인 슬로건: `분위기를 먼저 보고, 대화는 그 다음에`
- 보조 영문 방향: `Desire Ledger`

## 12. 운영 원칙

- 신고/차단은 빠르고 쉬워야 한다
- 미성년 의심, 협박, 금전 요구, 불법 콘텐츠는 최우선 대응
- 운영 판단은 일관되고 로그가 남아야 한다

## 13. 측정 원칙

- 성장보다 신뢰와 안정성을 먼저 본다
- 핵심 지표는 DM, 환불, 신고, 차단, 리텐션을 함께 본다
- 이벤트 트래킹은 상태 변화 중심으로 설계한다

## 14. 지금 당장 중요한 실행 우선순위

1. 상태도 기반 기능명세 유지
2. 프로필 정보 구조 강화
3. 포인트/환불 UX 고정
4. 운영 시나리오 정리
5. SQL / 마이그레이션 준비

## 15. 최종 판단

욕망백서의 핵심은 `자극적인 기능`이 아니라  
`탐색 -> 판단 -> 요청 -> 명확한 상태 변화` 구조다.

이 흐름이 유지되는 한, 문서와 구현은 같은 방향으로 갈 수 있다.
