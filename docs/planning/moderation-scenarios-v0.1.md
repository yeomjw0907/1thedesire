# 욕망백서 신고 / 차단 / 제재 시나리오 v0.1

## 1. 목적

- 운영정책을 실제 상황 단위로 풀어낸다.
- 대표 1인이 어떤 상황에서 무엇을 보고 어떤 조치를 해야 하는지 기준을 만든다.

## 2. 기본 원칙

- 위험도 높은 사안부터 우선 처리한다
- 법률 판단처럼 쓰지 않고 운영 판단 기준으로 정리한다
- 조치는 일관되어야 하고 로그가 남아야 한다
- 차단과 신고는 사용자 보호가 먼저다

## 3. 운영 레벨 정의

### Level 1

- 일반 분쟁
- 말투 불쾌감
- 가벼운 스팸

기본 조치:

- 신고 검토
- 경고 또는 기각

### Level 2

- 반복적 원치 않는 DM
- 외부 연락처 유도
- 금전 요구
- 노골적 괴롭힘

기본 조치:

- 계정 제한 또는 일시정지
- 관련 DM/게시글 확인

### Level 3

- 미성년 의심
- 협박
- 불법 촬영물/유포 시도
- 신상 유포
- 반복 사기

기본 조치:

- 즉시 제한 또는 영구정지
- 관련 기록 보존

## 4. 시나리오 1: 원치 않는 DM 반복 발송

### 상황

- 한 사용자가 여러 명에게 반복적으로 DM 요청을 보내고 차단을 많이 당함

### 운영자가 확인할 것

- 최근 DM 요청 수
- 거절률
- 차단 누적
- 신고 동시 발생 여부

### 조치 기준

- 경미하면 경고
- 반복되면 `restricted`
- 스팸 수준이면 `suspended`

### 사용자 화면 기대 동작

- 차단 즉시 상호 비노출
- 신고 접수 안내

## 5. 시나리오 2: 외부 연락처 유도

### 상황

- 카카오톡, 텔레그램, 오픈채팅 등 외부 이동을 집요하게 유도

### 운영자가 확인할 것

- 메시지 본문
- 동일 패턴 반복 여부
- 여러 신고에 같은 내용이 있는지

### 조치 기준

- 1회성 저강도: 경고 또는 제한
- 반복적/상업적 패턴: 일시정지
- 금전 목적 결합 시: 영구정지 검토

## 6. 시나리오 3: 금전 요구 또는 사기

### 상황

- 선입금 요구, 계좌 요구, 상품권 요구, 만남 조건부 금전 요구

### 운영자가 확인할 것

- 관련 채팅
- 포인트 분쟁 여부
- 동일 계정의 과거 이력

### 조치 기준

- 즉시 `suspended`
- 사기 패턴 명확 시 `banned`

### 추가 메모

- 포인트 환불 정책과 별개로 계정 제재가 먼저다

## 7. 시나리오 4: 협박 / 합의금 유도 / 압박

### 상황

- 신고하겠다, 돈을 보내라, 신상을 퍼뜨리겠다 등 압박

### 운영자가 확인할 것

- 대화 원문
- 차단 및 신고 시점
- 동일 패턴 반복 여부

### 조치 기준

- 즉시 `banned` 우선 검토
- 관련 기록 보존

## 8. 시나리오 5: 미성년 의심

### 상황

- 본인이 미성년처럼 언급
- 타 사용자가 미성년으로 의심 신고
- 프로필 또는 게시글이 미성년 맥락을 암시

### 운영자가 확인할 것

- 프로필
- 게시글
- 채팅 맥락

### 조치 기준

- 즉시 `suspended`
- 확인 전까지 노출 중단
- 명확하면 `banned`

## 9. 시나리오 6: 불법 촬영물 / 유출물 / 보복성 콘텐츠

### 상황

- 유출 사진 공유
- 상대 동의 없는 민감 이미지 전송
- 보복성 게시 시도

### 운영자가 확인할 것

- 게시글 또는 메시지 원문
- 첨부 여부
- 신고 다발 여부

### 조치 기준

- 즉시 `banned`
- 게시물 즉시 숨김 또는 삭제
- 관련 로그 보존

## 10. 시나리오 7: 일반적인 말투 분쟁

### 상황

- 무례함, 불쾌한 표현, 성향 불일치로 인한 감정 충돌

### 운영자가 확인할 것

- 실제 금지행위 해당 여부
- 반복성 여부

### 조치 기준

- 차단 권장
- 경고 또는 기각

## 11. 차단 시나리오 기준

### 사용자 차단이 발생하면

- 상호 프로필 노출 제한
- 추가 DM 요청 제한
- 현재 대화 상태 `blocked`
- 환불 없음

### 운영 메모

- 차단은 사용자 보호 행위이므로 별도 승인 없이 즉시 반영되는 것이 맞다

## 12. 신고 처리 상태 정의

- `open`: 접수됨
- `reviewing`: 검토중
- `resolved`: 조치 완료
- `dismissed`: 기각

## 13. 운영자 제재 액션 정의

- `warn`
- `restrict`
- `suspend`
- `ban`
- `hide_post`

## 14. 운영자 판단 체크리스트

- 이 사안이 반복적인가
- 여러 사용자에게 동시에 발생했는가
- 금전, 협박, 외부 이동 유도가 결합됐는가
- 미성년 또는 불법 콘텐츠 가능성이 있는가
- 즉시 차단/정지가 필요한가

## 15. 최종 판단

욕망백서 운영 시나리오의 핵심은 `빠른 보호`다.

애매한 상황을 오래 붙잡고 있기보다  
사용자 보호가 필요한 상황에선 먼저 제한하고, 그 다음 기록을 보는 쪽이 맞다.
