# 욕망백서 하이파이 UI 명세 v0.1

## 1. 목적

- 홈, 프로필, DM 핵심 화면을 실제 디자인 직전 수준으로 구체화한다.
- 레이아웃, 정보 우선순위, 시각 무드, 마이크로 인터랙션 기준을 고정한다.

## 2. 공통 하이파이 원칙

- 첫 인상은 자극이 아니라 신뢰다
- 모든 화면은 어둡지만 답답하지 않아야 한다
- 텍스트가 주인공이고 색은 보조다
- 과금과 상태는 절대 숨기지 않는다
- 20대 후반~30대 후반 사용자가 즉시 이해할 수 있어야 한다

## 3. 홈 화면 하이파이 명세

### 3-1. 화면 목적

- 유저가 서비스의 결을 빠르게 읽는다
- 사람을 카드가 아니라 글과 분위기로 탐색한다
- 프로필 이동과 글쓰기 둘 다 자연스럽게 유도한다

### 3-2. 화면 구조

```text
[Status Bar]

[Top App Bar]
좌: 워드마크 또는 심볼
우: 검색 아이콘 / 포인트 잔액

[Hero Strip]
짧은 문장 1줄
예: "분위기를 먼저 보고, 대화는 그 다음에"

[Segment Tabs]
전체 / 신규

[Feed List]
피드 카드 반복

[Floating Write Button]

[Bottom Navigation]
홈 / 검색 / DM / 포인트 / 프로필
```

### 3-3. 시각 스타일

- 배경은 `bg-900`
- 상단 바는 완전 분리하지 않고 배경과 자연스럽게 섞이게 함
- Hero Strip은 카드형이 아니라 얇은 editorial strip처럼 처리
- 피드 카드는 레이어가 겹쳐 보이도록 surface 대비를 줌

### 3-4. 피드 카드 하이파이 구조

```text
[메타 행]
닉네임
연령대 · 지역
성향 태그 1~2개
작성 시간

[본문]
최대 4줄 노출

[선택 이미지]
있을 경우 낮은 채도 톤으로 정리

[하단 행]
프로필 보기
신고
```

### 3-5. 마이크로 인터랙션

- 카드 탭 시 scale보다 opacity와 elevation 변화 중심
- 스크롤 시 상단 바는 약하게 블러 또는 반투명 처리
- 글쓰기 버튼은 크게 튀지 않고 천천히 나타나는 방식

### 3-6. 홈 화면에서 절대 하지 말 것

- 하트 수, 랭킹, 등급을 과하게 전면에 두지 않기
- 자극적인 대표 이미지 크게 깔지 않기
- 포인트 충전 CTA를 홈에서 과도하게 밀지 않기

## 4. 프로필 화면 하이파이 명세

### 4-1. 화면 목적

- 이 사람에게 DM을 보낼지 말지 빠르게 판단하게 한다
- 외모보다 분위기, 취향, 신뢰를 먼저 읽게 한다

### 4-2. 화면 구조

```text
[Top Bar]
뒤로가기 / 신고 / 차단

[Profile Header]
닉네임
연령대 · 지역 · 성별
성향

[Trust Line]
최근 활동
DM 가능 상태
여성일 경우: "받은 요청 수락 무료"

[Bio Section]
자기소개 본문

[Recent Posts Section]
최근 글 카드 3~5개

[Sticky Bottom CTA]
대화 요청 보내기
정책 요약 2줄
```

### 4-3. 시각 스타일

- 프로필은 홈보다 더 정돈된 화면이어야 함
- 정보 블록 간 여백을 넉넉히 줘서 읽기 쉽게 구성
- 헤더는 사진이 없어도 성립해야 함
- 있더라도 사진은 주인공이 아니라 보조 요소

### 4-4. DM CTA 영역 하이파이 구성

```text
[Primary CTA]
대화 요청 보내기

[Rule Summary]
요청 시 90P 차감
거절 시 45P 환불 · 미응답 시 전액 환불
```

### 4-5. 마이크로 인터랙션

- 최근 글 카드 탭 시 부드러운 전환
- DM 버튼 탭 시 즉시 바텀시트 상승
- 차단/신고는 확인 시트로 한 번 더 확인

### 4-6. 프로필 화면에서 절대 하지 말 것

- 사진을 지나치게 크게 두지 않기
- 자기소개보다 액션 버튼이 먼저 보이게 하지 않기
- 과도한 장식이나 금속 질감 UI 쓰지 않기

## 5. DM 요청 팝업 하이파이 명세

### 5-1. 화면 목적

- 사용자가 손해 구조가 아니라는 점을 바로 이해하게 한다
- 결제가 아니라 요청 절차로 인식되게 만든다

### 5-2. 구조

```text
[Sheet Handle]

[Title]
대화 요청 보내기

[Target Summary]
닉네임
연령대 · 지역
성향

[Policy Panel]
90P 차감
거절 시 45P 환불
24시간 미응답 시 전액 환불
수락 후 무료 대화

[Action Row]
취소 / 요청 보내기
```

### 5-3. 시각 스타일

- 바텀시트는 매트한 surface 사용
- 정책 패널은 한 단계 밝은 surface로 구분
- 숫자 `90P`, `45P`는 Montserrat로 강조

## 6. DM 목록 화면 하이파이 명세

### 6-1. 화면 목적

- 요청 상태와 진행 중 대화를 빠르게 파악하게 한다

### 6-2. 구조

```text
[Top Bar]
DM

[Tabs]
대기 / 진행중 / 종료

[List]
상태 카드 반복
```

### 6-3. 카드 구조

```text
닉네임
상태 배지
보조 설명
시간 정보
```

예:

- 응답 대기중 / 상대의 응답을 기다리는 중입니다
- 거절됨 / 45P가 환불되었습니다
- 만료됨 / 90P가 환불되었습니다

## 7. 채팅방 하이파이 명세

### 7-1. 화면 목적

- 상태를 분명히 유지하면서도 대화에 집중하게 한다

### 7-2. 구조

```text
[Top Bar]
뒤로가기
상대 닉네임
프로필 보기
신고
차단

[State Banner]
현재 상태 설명

[Message List]
시스템 메시지
사용자 메시지

[Input Area]
상태에 따라 활성/비활성
```

### 7-3. 시각 스타일

- 채팅은 가장 기능적이어야 한다
- 시스템 메시지는 중앙 정렬, 작은 배지형
- 사용자 메시지는 radius가 큰 부드러운 버블
- 비활성 입력창도 이유를 읽을 수 있게 구성

### 7-4. 상태 배너 문구 예시

- 상대의 응답을 기다리는 중입니다
- 대화를 시작할 수 있습니다
- 요청이 거절되어 45P가 환불되었습니다
- 요청이 만료되어 90P가 환불되었습니다
- 더 이상 대화를 이어갈 수 없습니다

## 8. 모바일 최적화 기준

- 주요 CTA는 엄지 영역 하단에 배치
- 상단 액션은 2개 이내
- 탭 라벨은 아이콘과 함께 노출
- 본문 글씨는 15px 이상 유지

## 9. 디자인 QA 체크리스트

- 이 화면이 선정적이지 않고도 매력적인가
- 글과 정보가 충분히 읽히는가
- 사용자가 현재 상태를 즉시 이해하는가
- 과금/환불 구조가 숨겨지지 않는가
- 30대 사용자에게 유치하게 느껴지지 않는가

## 10. 최종 판단

욕망백서의 하이파이 UI는 `다크 모드 성인 서비스`가 아니라 `은밀한 관계 편집지`처럼 보여야 한다.

피드, 프로필, DM 모두 그 공통 감각을 유지해야 브랜드가 성립한다.
