# 욕망백서 핵심 화면 UX 원칙 v0.1

## 1. 목적

- 홈, 프로필, DM 화면에서 욕망백서다운 UX 기준을 정한다.
- 경쟁 서비스에서 참고한 요소를 실제 화면 원칙으로 변환한다.

## 2. 공통 UX 원칙

### 2-1. 자극보다 신뢰를 먼저 보여준다

- 첫 화면부터 과한 노출 이미지나 선정적 문구를 앞세우지 않는다.
- 서비스가 `어떤 분위기의 사람들을 위한 공간인지`를 먼저 전달한다.

### 2-2. 익명성은 기능이 아니라 경험이다

- 이름, 위치, 이미지 노출은 항상 통제 가능해야 한다.
- 유저는 자신의 노출 수준을 스스로 조절한다고 느껴야 한다.

### 2-3. 행동에는 항상 맥락을 붙인다

- DM 요청, 포인트 차감, 환불, 차단은 결과만 보여주면 안 된다.
- 왜 차감되는지, 어떤 상태인지, 다음엔 어떻게 되는지가 보여야 한다.

### 2-4. 분위기 탐색이 먼저다

- 바로 들이대는 구조보다, 피드와 프로필을 통해 상대 분위기를 읽게 해야 한다.
- 욕망백서는 매칭 앱보다는 `분위기 읽고 들어가는 앱`이어야 한다.

## 3. 홈 화면 UX 원칙

### 3-1. 홈의 역할

- 홈은 콘텐츠 소비 화면이 아니라 `사람의 분위기를 읽는 화면`이다.

### 3-2. 홈에서 보여줘야 할 것

- 글 내용
- 작성자 분위기
- 지역/성향/태그
- 최근성

### 3-3. 홈에서 숨겨야 할 것

- 과도한 구매 유도
- 지나치게 노골적인 성적 문구
- 과한 배지와 등급 정보

### 3-4. 추천 UX 규칙

- 게시글 카드에서 프로필로 자연스럽게 이동되어야 한다.
- 글쓰기 버튼은 분명해야 하지만 피드를 압도하면 안 된다.
- 필터는 복잡하게 시작하지 말고 `전체 / 신규` 정도로 얕게 시작한다.
- 첫 세션에서 피드 밀도를 느끼게 해야 하므로 자동 생성 게시글도 흐름을 만드는 역할을 해야 한다.

### 3-5. 홈의 성공 기준

- 스크롤이 자연스럽다
- 글 하나를 보고 프로필로 이동하고 싶어진다
- 피드가 싸구려 성인 사이트처럼 보이지 않는다

## 4. 프로필 화면 UX 원칙

### 4-1. 프로필의 역할

- 프로필은 자기소개 카드가 아니라 `대화 시작 전 판단 화면`이다.

### 4-2. 프로필에서 먼저 보여줄 것

- 닉네임
- 연령대
- 지역
- 성향
- 자기소개
- 최근 게시글

### 4-3. 프로필에서 중요하게 다뤄야 할 것

- 외모보다 취향
- 사진보다 분위기
- 소개보다 신뢰감

### 4-4. DM 버튼 설계 원칙

- DM 버튼은 항상 보이되, 섣부른 압박감을 주면 안 된다.
- 버튼 근처에 `90P 차감`, `거절 시 50% 환불`, `미응답 만료 시 전액 환불` 같은 핵심 규칙을 짧게 보여주는 것이 좋다.
- 여성 유저 프로필에서는 `받은 DM 수락은 무료` 같은 안전/메리트 메시지를 별도로 보여줄 수 있다.

### 4-5. 프로필의 성공 기준

- 유저가 이 사람과 대화할지 말지 빠르게 판단할 수 있다
- DM 전에 불안감이 줄어든다
- 신고/차단 버튼이 필요할 때 쉽게 보인다

## 5. DM 요청 UX 원칙

### 5-1. DM의 역할

- DM은 과금 액션이 아니라 `합의 요청 액션`으로 인식되어야 한다.

### 5-2. DM 요청 팝업에서 보여줘야 할 것

- 차감 포인트
- 거절 시 환불
- 미응답 만료 규칙
- 수락 후 무료 대화 가능 여부

### 5-3. 문구 원칙

- 짧고 명확해야 한다
- 겁주지 말고 설명해야 한다
- 법적 문구보다 서비스 규칙을 먼저 보여줘야 한다

예시:

- 요청 시 90P가 차감됩니다
- 상대가 거절하면 45P가 환불됩니다
- 24시간 내 응답이 없으면 전액 환불됩니다
- 수락 후 대화는 무료입니다

### 5-4. 대기 상태 UX 원칙

- 대기중, 수락, 거절, 만료 상태가 한눈에 구분돼야 한다
- 유저는 현재 요청이 어디까지 갔는지 알 수 있어야 한다
- 환불이 발생한 경우 즉시 내역이 반영돼야 한다

### 5-5. 채팅방 UX 원칙

- 수락 전에는 입력창을 잠그고 이유를 설명한다
- 수락 후에는 과금 걱정 없이 바로 대화에 집중할 수 있어야 한다
- 신고와 차단은 접근이 쉬워야 하지만 대화를 방해하지는 않아야 한다

## 6. 경쟁 서비스에서 가져온 UX 포인트

### BLUV에서 참고할 점

- 상세 프로필
- 신뢰 메시지
- 국내 유저가 이해하기 쉬운 구조

### PURE에서 참고할 점

- 익명성 중심 카피
- 부담 없는 첫 진입
- 프라이버시를 체감하게 하는 UX

### Feeld에서 참고할 점

- 안전 메시지의 문서화
- 현대적이고 절제된 시각 언어

### FetLife에서 참고할 점

- 커뮤니티의 밀도
- 취향을 설명하는 언어

## 7. 욕망백서만의 UX 방향

### 홈

- `트위터처럼 가볍게 보이되, 더 정제된 피드`

### 프로필

- `소개팅 카드가 아니라 취향 보고서`

### DM

- `결제 팝업이 아니라 합의 요청 단계`

## 8. 디자인팀 또는 구현팀 체크리스트

- 이 화면이 신뢰를 먼저 주는가
- 이 화면이 유저의 노출 통제감을 높이는가
- 이 화면이 상대의 분위기를 읽게 만드는가
- 이 화면이 과금 규칙을 숨기지 않는가
- 이 화면이 저질스럽게 보이지 않는가

## 9. 최종 판단

욕망백서 UX의 핵심은 `빠른 유혹`이 아니라 `안심되는 진입`이다.

유저는 여기서 바로 소비되고 싶어 하지 않는다.  
먼저 분위기를 보고, 상대를 읽고, 설명을 이해한 뒤, 그 다음에 대화를 시작하고 싶어 한다.

그 흐름을 방해하지 않는 것이 욕망백서 UX의 핵심이다.
