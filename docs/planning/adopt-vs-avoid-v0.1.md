# 욕망백서 차용 기능 / 배제 기능 실행 문서 v0.1

## 1. 목적

- 해외 및 국내 벤치마크 조사 결과를 실제 제품 판단 기준으로 내린다.
- 무엇을 가져오고 무엇을 버릴지 실행 관점에서 고정한다.

## 2. 판단 기준

아래 세 가지 중 두 개 이상에 긍정적이면 차용 대상으로 본다.

- 커뮤니티 밀도를 높이는가
- 대화 전 불안을 줄이는가
- 브랜드를 더 세련되고 신뢰감 있게 만드는가

반대로 아래 중 두 개 이상이면 배제 대상으로 본다.

- 저가형 성인앱처럼 보이게 만드는가
- 제품 구조를 불필요하게 무겁게 만드는가
- 신뢰를 떨어뜨리거나 운영 리스크를 키우는가

## 3. 반드시 차용할 기능

### 3-1. 피드 중심 탐색

출처:

- FetLife
- BLUV

왜 차용하는가:

- 사람을 매칭 카드가 아니라 `글과 분위기`로 먼저 읽게 만든다.
- 체류시간과 커뮤니티 밀도에 직접 연결된다.

욕망백서 적용 방식:

- 홈 피드를 메인 진입 화면으로 둔다.
- 자동 생성 소개글도 피드 밀도 확보 장치로 사용한다.

### 3-2. 프로필을 판단 도구로 만드는 구조

출처:

- BLUV
- Feeld

왜 차용하는가:

- 유저가 DM 전에 상대를 판단할 근거가 필요하다.
- 단순 사진 카드보다 신뢰도가 높다.

욕망백서 적용 방식:

- 닉네임, 연령대, 지역, 성향, 자기소개, 최근 게시글을 묶어 보여준다.
- 프로필은 `소개팅 카드`가 아니라 `취향 보고서`처럼 설계한다.

### 3-3. 새 DM 요청만 과금, 연결 후 무료 대화

출처:

- Feeld의 연결 후 대화 구조
- PURE의 부담 낮은 연결 UX
- Ashley Madison의 접촉 시점 과금 감각

왜 차용하는가:

- 과금 포인트가 명확하다.
- 연결 이후 추가 과금이 없으면 대화 이탈이 줄어든다.

욕망백서 적용 방식:

- 새 DM 요청 90P
- 거절 45P 환불
- 24시간 미응답 전액 환불
- 수락 후 무료 대화

### 3-4. 여성 유저 중심의 안전/메리트 메시지

출처:

- PURE
- Ashley Madison의 여성 유입 전략

왜 차용하는가:

- 성인 취향 플랫폼은 여성 유저의 심리적 안전감이 특히 중요하다.
- 여성 유입이 전체 매칭 품질과 서비스 분위기에 직접 영향을 준다.

욕망백서 적용 방식:

- 여성 가입 시 270P 지급
- 받은 DM 수락 무료
- 차단과 신고 진입성을 높임
- 카피에서 `통제감`을 강조

### 3-5. 프로필 공개 제어

출처:

- BLUV
- Feeld

왜 차용하는가:

- 익명성은 단순히 닉네임만 숨긴다고 생기지 않는다.
- 사용자가 노출 수준을 통제할 수 있어야 심리적 안전감이 생긴다.

욕망백서 적용 방식:

- MVP 이후 우선 확장 기능으로 둔다.
- 사진/상세정보 공개 범위를 추후 옵션화한다.

### 3-6. 안전 정책의 전면 문서화

출처:

- Feeld
- PURE

왜 차용하는가:

- 안전은 기능보다 먼저 메시지로 보여야 한다.
- 신뢰는 약관에만 있으면 늦다.

욕망백서 적용 방식:

- 신고, 차단, 환불, 금지행위를 문서와 UI 양쪽에 짧고 명확하게 노출한다.

## 4. 조건부 차용 기능

### 4-1. 취향 태그의 구조화

출처:

- Feeld
- FET 계열

조건:

- 첫 가입 단계에서 너무 무겁지 않게 설계할 것
- 자유입력과 구조화 태그를 균형 있게 사용할 것

### 4-2. 캡처 방지 또는 워터마크

출처:

- BLUV 업데이트 포인트

조건:

- 기술 비용 대비 효과를 검토할 것
- UX를 과하게 망치지 않을 것

### 4-3. 오프라인 만남 안전 기능

출처:

- PURE Safety Signal

조건:

- MVP 이후 확장 기능으로 둘 것
- 운영 리스크와 실제 사용성을 같이 검토할 것

## 5. 명확히 배제할 기능

### 5-1. 과도한 그룹/이벤트 중심 구조

출처:

- FetLife

왜 배제하는가:

- 초반 MVP를 무겁게 만든다.
- 한국형 첫 진입에 너무 복잡하다.

### 5-2. 구인앱처럼 보이는 라이브 모집 구조

출처:

- 일부 국내 BDSM 앱

왜 배제하는가:

- 서비스가 급격히 저가형/위험한 인상으로 떨어진다.
- 브랜드 톤과 충돌한다.

### 5-3. 과한 등급제/게임화

출처:

- 일부 국내 유사앱

왜 배제하는가:

- 성숙한 연령층에 맞지 않는다.
- 서비스 신뢰보다 중독 설계가 전면에 나와 보인다.

### 5-4. 훅업 중심 자극 카피

출처:

- 일부 글로벌 훅업 앱

왜 배제하는가:

- 앱스토어 리스크와 별개로 브랜드 질감이 무너진다.
- 여성 유저 안전감과 충돌한다.

### 5-5. 복잡한 가입 단계 분류

출처:

- 일부 글로벌 관계 탐색 앱

왜 배제하는가:

- 초반 이탈이 커진다.
- 욕망백서는 `빠른 진입 후 천천히 탐색`이 더 맞다.

### 5-6. 실제 사람 여부를 흐리는 구조

출처:

- 일부 문제적 약관/유사 서비스

왜 배제하는가:

- 신뢰가 즉시 붕괴한다.
- 운영/법률 리스크가 커진다.

## 6. 제품에 바로 반영할 실행안

### 바로 반영

- 홈 피드 중심 구조 유지
- 프로필을 판단 도구로 강화
- DM 상태와 환불 규칙 노출 강화
- 여성 메리트 메시지 강화
- 안전 정책의 UI 내 노출

### 다음 단계 반영

- 성향 태그 구조화
- 프로필 공개 범위 제어
- 최근 활동 표시
- DM 받기 상태 표시

### 보류

- 캡처 방지
- 오프라인 안전 기능
- 그룹/이벤트 구조

## 7. 최종 판단

욕망백서는 기능을 많이 가져오는 방식으로 커지지 않는다.

핵심은 아래를 정확히 가져오는 것이다.

- FetLife의 커뮤니티성
- Feeld의 구조적 세련됨
- PURE의 익명성과 통제감
- Ashley Madison식 접촉 시점 과금 감각

그리고 아래는 끝까지 버려야 한다.

- 구인앱 느낌
- 싸구려 성인앱 카피
- 과한 게임화
- 신뢰를 무너뜨리는 구조
