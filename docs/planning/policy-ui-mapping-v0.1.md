# 욕망백서 운영정책 + UI 연결 문서 v0.1

## 1. 목적

- 운영정책이 실제 제품 UI 어디에서 어떻게 노출되어야 하는지 연결한다.
- 정책은 문서에만 존재하지 않고 사용자 경험 안에서 보이도록 설계한다.

## 2. 기본 원칙

- 정책은 약관에만 숨기지 않는다
- 사용자가 행동하기 직전에 필요한 정책을 보여준다
- 짧고 명확한 문구로 UI에 녹인다
- 과도한 경고보다 통제감과 안내 중심으로 표현한다

## 3. 정책과 UI의 연결 필요성

욕망백서는 아래 항목이 신뢰를 좌우한다.

- DM 과금
- 환불
- 차단
- 신고
- 금지행위
- 성인 확인

이 정보들이 행동 직전에 안 보이면 불신이 생긴다.

## 4. 가입 정책과 UI 연결

### 정책

- 19세 이상 체크 필수
- 약관 및 운영정책 동의 필수
- 자동 소개글 생성

### 노출 위치

- 가입 동의 화면
- 가입 완료 화면

### UI 문구 예시

- 본인은 19세 이상입니다
- 운영정책 및 이용약관에 동의합니다
- 첫 소개글이 등록되었습니다. 언제든 수정하거나 삭제할 수 있습니다

## 5. DM 과금 정책과 UI 연결

### 정책

- 새 DM 요청 시 90P 차감
- 거절 시 45P 환불
- 24시간 미응답 시 전액 환불
- 수락 후 무료 대화

### 노출 위치

- 프로필 하단 CTA 영역
- DM 요청 팝업
- DM 목록 상태 카드
- 채팅방 상태 배너
- 포인트 화면

### UI 문구 예시

- 요청 시 90P가 차감됩니다
- 거절 시 45P가 환불됩니다
- 24시간 내 응답이 없으면 전액 환불됩니다
- 수락 후 대화는 무료입니다

## 6. 차단 정책과 UI 연결

### 정책

- 차단 시 상호 프로필/대화 제한
- 차단 시 환불 없음

### 노출 위치

- 차단 확인 시트
- 차단 완료 토스트
- DM 상태 화면

### UI 문구 예시

- 차단하면 서로의 프로필과 대화가 제한됩니다
- 차단된 요청은 환불되지 않습니다
- 더 이상 이 대화를 이어갈 수 없습니다

## 7. 신고 정책과 UI 연결

### 정책

- 프로필, 게시글, 채팅 신고 가능
- 협박, 외부 연락처 요구, 금전 요구, 미성년 의심 등은 신고 대상

### 노출 위치

- 프로필 상단 액션
- 게시글 카드
- 채팅방 상단
- 도움말 또는 운영정책 요약 화면

### UI 문구 예시

- 불편하거나 부적절한 행동이 있었다면 알려주세요
- 외부 연락처 요구, 금전 요구, 협박은 신고 대상입니다
- 신고가 접수되었습니다

## 8. 금지행위 정책과 UI 연결

### 정책

- 미성년 관련 내용 금지
- 협박, 사기, 사칭, 불법 촬영물 금지

### 노출 위치

- 가입 시 동의 화면
- 운영정책 요약 페이지
- 신고 시트 하단 보조 문구

### UI 문구 예시

- 이 공간은 취향을 존중하지만, 타인의 경계를 침범할 수는 없습니다
- 미성년 관련 내용, 협박, 사칭, 유출물 공유는 금지됩니다

## 9. 여성 메리트 정책과 UI 연결

### 정책

- 여성 가입 시 270P 지급
- 받은 DM 수락 무료

### 노출 위치

- 가입 완료 화면
- 프로필 보조 문구
- 포인트 화면

### UI 문구 예시

- 가입 보너스로 270P가 지급되었습니다
- 받은 요청을 수락하고 대화하는 데에는 포인트가 들지 않습니다

## 10. 운영 신뢰를 만드는 UI 포인트

- 상태를 숨기지 않는다
- 환불이 일어나면 바로 보여준다
- 정책은 행동 직전에 보인다
- 신고와 차단은 찾기 쉬운 위치에 둔다

## 11. UI에 과하게 노출하면 안 되는 것

- 법률 문구를 과도하게 길게 쓰기
- 경고문을 모든 화면에 반복 노출하기
- 정책 설명이 메인 액션을 가리게 하기

## 12. 추천 정책 노출 우선순위

### 항상 보여야 하는 것

- DM 과금 / 환불 요약
- 신고 / 차단 접근

### 특정 순간에만 보여야 하는 것

- 가입 동의 정책
- 자동 소개글 안내
- 여성 메리트 안내

### 도움말 또는 정책 페이지에 두면 되는 것

- 금지행위 전체 목록
- 운영 기준 상세

## 13. 최종 판단

욕망백서에서 정책은 `백오피스 문서`가 아니라  
사용자가 신뢰를 느끼게 하는 UX 요소다.

좋은 정책은 길게 적는 것이 아니라, 필요한 순간에 정확히 보이게 하는 것이다.
