# 서비스 프로세스 전반 개선 제안 v0.1

전체 플로우(진입 → 가입 → 홈 → 검색/DM/포인트 → 예외 처리)를 점검한 뒤, 우선순위를 두고 정리한 개선 제안입니다.  
**기억해 둘 변경**: 성향 = 프로필 Dom/Sub/Switch만, FWB·데이팅 등 = 글 쓸 때 태그 (별도 구현 예정).

---

## 1. 진입·로그인

| 현황 | 제안 | 우선순위 |
|------|------|----------|
| 루트(/)는 비로그인 → /login, 로그인+프로필 없음 → /signup, 있음 → /home만 수행 | 명세의 "보조 CTA 분위기 둘러보기"가 없음. **비로그인 시** 로그인 페이지에 "가입 후 분위기를 둘러볼 수 있어요" 문구가 이미 있음. 필요 시 "예시 피드 1~2장" 비로그인 미리보기 페이지 검토 | 낮음 |
| 로그인 에러(no_code, auth_failed)는 로그인 페이지에 표시됨 | 매직 링크 도입 시 /auth/confirm 에러(만료, 잘못된 링크 등) 문구 추가 | 매직 링크 시 |

---

## 2. 가입

| 현황 | 제안 | 우선순위 |
|------|------|----------|
| 성향 = 그룹별 칩 다수 (역할·취향·관계) | **예정**: 역할 = Dom/Sub/Switch 1개만, FWB 등 = 글 태그로 이전 | 이미 계획됨 |
| 가입 완료 화면에 "홈 피드 보기"만 있음 | first-7-days 명세의 Day 0 "내 프로필 확인 유도"를 위해 **"내 프로필 보기"** 보조 버튼 또는 링크 추가 시, 첫 방문 경로가 명확해짐 | 중간 |
| 닉네임 중복 등 에러 시 폼 상태 초기화 | 서버 액션 반환 시 기존 입력값 복원(role 등)은 나중 단계에서 검토. 문서화만 해두어도 됨 | 낮음 |

---

## 3. 홈·피드

| 현황 | 제안 | 우선순위 |
|------|------|----------|
| 빈 피드 시 EmptyState 존재 | 문구가 "아직 올라온 분위기가 없어요" 등인지 확인 후, **"첫 분위기를 남겨보세요"** 등 행동 유도 한 줄 추가 권장 | 낮음 |
| 피드 카드에 작성자 role(태그) 표시 | 역할 단순화 후에는 프로필 role 1개 + (도입 시) **글 태그** 표시로 정리 | 태그 도입 시 |
| 홈 헤더에 포인트·글쓰기 노출 | 현재 구조 유지. 포인트 부족 시 유도는 DM 요청 시점에서 처리(아래 참고) | - |

---

## 4. 글쓰기 (분위기 남기기)

| 현황 | 제안 | 우선순위 |
|------|------|----------|
| content + image 만 수집, **태그 없음** | **posts.tags** 컬럼 추가 후, 글쓰기 폼에 태그 입력 추가 (FWB, 감성 연애, 대화 위주, 만남 위주 등 제안 + 직접 입력). 성향·태그 계획과 일치 | 높음 (성향 정리와 함께) |
| 10자 최소·300자 제한, 에러 메시지 표시 | 유지 | - |

---

## 5. 검색

| 현황 | 제안 | 우선순위 |
|------|------|----------|
| QUICK_ROLES = Dom, Sub, Switch, FWB, 감성 연애, 대화 위주 (프로필 role 기반 검색) | 역할 단순화 후 **역할 필터**는 Dom/Sub/Switch만. FWB·감성 연애 등은 **글 태그** 검색으로 이전 시 QUICK_ROLES를 역할 3개 + (선택) 인기 글 태그로 정리 | 태그 도입 시 |
| 검색어 없이 진입 시 EmptyPrompt | "지역·성향을 골라보거나 검색어를 입력해보세요" 등 한 줄 안내로 일관성 유지 | 낮음 |
| 프로필 탭 / 분위기 탭 | 분위기 탭은 **글** 검색. posts.tags 도입 후 태그 필터 연동 | 태그 도입 시 |

## 6. DM

| 현황 | 제안 | 우선순위 |
|------|------|----------|
| 포인트 부족 시 시트 내 "포인트가 부족합니다. 현재 N P 보유"만 표시 | **"포인트 충전하러 가기"** 링크 또는 버튼 추가 → /points 로 유도. 전환 경로 명확해짐 | 높음 |
| 요청 시 90P, 거절 45P 환불, 만료 전액 환불 안내 | 현재 문구 유지 | - |
| DM 목록 탭(대기/진행중/종료), 만료 남은 시간 표시 | 유지 | - |
| 수신자 24시간 미응답 시 만료 | cron 등으로 만료 처리되는지 이미 구현됨. UX 측면에서는 "만료됨" 상태 문구만 확인 | - |

## 7. 포인트

| 현황 | 제안 | 우선순위 |
|------|------|----------|
| 무통장 입금 안내, 입금자명=닉네임, 1~2시간 내 처리, support@ 문의 | 현재 플로우 유지. 자동 입금 확인·결제 연동은 별도 과제 | - |

| 현황 | 제안 | 우선순위 |
|------|------|----------|

| 타인 프로필에서 DM 요청, 신고, 차단 | 유지 | - |

## 9. 에러·예외 처리

| 현황 | 제안 | 우선순위 |
|------|------|----------|
| app/not-found.tsx 없음 | **404 전역 페이지** 추가 (간단한 문구 + 홈/로그인 링크). 잘못된 URL·삭제된 방 등에서 일관된 UX | 중간 |
| profile/[id] notFound(), dm/[roomId] notFound() | 유지 | - |
| 미들웨어: 비인증 → /login, 프로필 없음 → /signup | /auth/confirm 허용만 매직 링크 도입 시 추가하면 됨 | 매직 링크 시 |

## 10. 일관성·정리

| 항목 | 제안 |
|------|------|
| **역할·태그 정책** | 프로필 role = Dom/Sub/Switch만. 검색 역할 필터, 프로필 수정, 가입 폼, 피드/프로필 카드 표시를 이에 맞춰 통일. FWB·데이팅 등은 글 태그로만 사용. |
| **문서** | onboarding-signup-spec, first-7-days-experience, role-bdsm-ux-plan, role-tags-ux-comparison 반영 후, 이 문서(service-flow-improvement-suggestions)를 "구현 전 체크리스트"처럼 업데이트해 두면 좋음. |

## 11. 우선순위 요약

- **높음 (기능·전환과 직결)**  
  - DM 요청 시 포인트 부족 → "포인트 충전하러 가기" 유도  
  - 글쓰기 태그 도입 (posts.tags + UI)  
  - 성향 단순화(가입·프로필·검색) + 글 태그 정책 적용  

- **중간 (경험 보완)**  
  - 가입 완료 화면에 "내 프로필 보기" 보조 CTA  
  - 전역 404(not-found) 페이지  

- **낮음 (여유 있을 때)**  
  - 빈 피드 EmptyState 문구 강화  
  - 검색 EmptyPrompt 문구  
  - 포인트 내역 라벨 보강  
  - 가입 에러 시 폼 복원  

- **다른 기능과 함께**  
  - 매직 링크: /auth/confirm 에러 메시지, 미들웨어 예외  
  - 역할 단순화와 동시: 검색 QUICK_ROLES, 프로필 수정 role, 피드/카드 표시  

이 순서로 적용하면 서비스 프로세스가 한 번에 정리되고, 이후 확장(결제 연동, 푸시 등)도 붙이기 쉬워집니다.
