# 욕망백서 개발 태스크 백로그 v0.1

## 1. 개발 원칙

- MVP 범위 안에서만 구현한다.
- 상태값과 로그를 먼저 설계하고 UI를 붙인다.
- 운영 정책이 미정인 항목은 코드에서 확장 가능하게 만든다.
- 모바일 우선으로 화면을 설계한다.

## 2. 우선순위 기준

- P0: 서비스 핵심 흐름이 막히는 항목
- P1: 출시 품질과 운영 안정성을 높이는 항목
- P2: 출시 후 개선 가능한 항목

## 3. 에픽 구성

### 에픽 A. 공통 인프라

목표:

- 프로젝트 기본 구조, 인증 연동, 공통 UI 기반을 만든다

태스크:

- P0 프로젝트 초기 세팅
- P0 환경변수 구조 정의
- P0 Supabase 연결 설정
- P0 공통 레이아웃과 모바일 뷰포트 기준 정의
- P0 라우팅 구조 설계
- P1 공통 에러 처리 방식 정의
- P1 로딩 및 빈 상태 UI 컴포넌트 제작

### 에픽 B. 인증 및 가입

목표:

- 유저가 가입하고 프로필을 생성할 수 있어야 한다

태스크:

- P0 간편 로그인 연동
- P0 가입 완료 여부 판별 로직
- P0 가입 폼 구현
- P0 필수 입력값 검증
- P0 `19세 이상입니다` 체크 처리
- P0 약관 동의 처리
- P0 자기소개 입력 필수 처리
- P0 프로필 생성 DB 연동
- P0 여성 회원 270P 가입 보너스 지급 로직
- P0 가입 완료 후 자동 소개글 생성 로직
- P1 닉네임 중복 검사
- P1 가입 실패 케이스 메시지 정리

완료 기준:

- 신규 유저가 로그인 후 가입 폼을 제출하면 프로필과 자동 게시글이 생성된다

### 에픽 C. 홈 피드 및 글쓰기

목표:

- 유저가 피드를 보고 글을 작성할 수 있어야 한다

태스크:

- P0 홈 피드 목록 조회
- P0 최신순 정렬
- P0 게시글 카드 UI
- P0 글쓰기 화면 구현
- P0 게시글 생성 API 또는 액션 구현
- P0 이미지 1장 업로드 처리
- P0 게시글 삭제 기능
- P1 게시글 수정 기능
- P1 자동 생성 게시글 구분 필드 반영
- P1 빈 피드 상태 UI

완료 기준:

- 유저가 홈에서 글 목록을 보고 직접 게시글을 작성 및 삭제할 수 있다

### 에픽 D. 검색 및 프로필 탐색

목표:

- 유저가 관심 있는 상대를 탐색할 수 있어야 한다

태스크:

- P0 프로필 상세 조회
- P0 내가 아닌 유저 프로필 이동
- P0 닉네임 검색
- P1 지역 필터
- P1 성향 필터
- P1 태그 검색
- P1 검색 결과 카드 UI

완료 기준:

- 유저가 검색을 통해 상대 프로필로 이동할 수 있다

### 에픽 E. DM 요청 및 합의

목표:

- 유저가 포인트를 사용해 DM 요청을 보내고, 수락 후 채팅을 시작할 수 있어야 한다

태스크:

- P0 DM 요청 팝업 구현
- P0 보유 포인트 확인 로직
- P0 90P 즉시 차감 처리
- P0 chat_rooms 생성
- P0 consent_events 기록
- P0 DM 요청 상태 표시
- P0 상대방 수락 기능
- P0 상대방 거절 기능
- P0 거절 시 45P 환불 처리
- P0 미응답 만료 시 90P 전액 환불 처리
- P0 차단 시 환불 없음 처리
- P0 수락 전 입력창 비활성 처리
- P0 수락 후 채팅 활성화
- P0 24시간 만료 처리 배치 또는 스케줄 로직
- P1 만료 상태 UI
- P1 환불 안내 문구 정리

완료 기준:

- DM 요청, 수락, 거절, 부분 환불 흐름이 일관되게 동작한다

### 에픽 F. 채팅

목표:

- 합의 완료된 유저만 1:1 채팅을 할 수 있어야 한다

태스크:

- P0 채팅방 목록 조회
- P0 채팅 메시지 조회
- P0 메시지 전송
- P0 시스템 메시지 처리
- P0 차단 후 메시지 차단 처리
- P1 종료된 방 상태 표시
- P1 마지막 메시지 시간 표시

완료 기준:

- 합의 완료 방에서만 메시지가 오가고 상태가 목록에 반영된다

### 에픽 G. 포인트 및 결제

목표:

- 포인트 잔액, 충전, 사용 내역을 관리할 수 있어야 한다

태스크:

- P0 포인트 잔액 표시
- P0 point_transactions 테이블 연동
- P0 충전 내역 조회
- P0 차감 및 환불 내역 조회
- P0 입금 안내 화면 구성
- P1 payment_events 저장 구조 구현
- P1 입금 확인 자동화 훅 연결
- P1 충전 완료 후 포인트 반영

완료 기준:

- 유저가 잔액과 거래 내역을 확인할 수 있고, 포인트 차감과 환불 기록이 남는다

### 에픽 H. 신고 및 차단

목표:

- 유저 스스로 위험한 상대를 차단하고 운영자에게 신고할 수 있어야 한다

태스크:

- P0 프로필 신고
- P0 게시글 신고
- P0 채팅 신고
- P0 유저 차단
- P0 차단 관계 저장
- P0 차단 시 프로필 및 DM 비노출 처리
- P1 신고 사유 선택지 설계
- P1 신고 상세 내용 입력

완료 기준:

- 신고와 차단이 유저 단에서 즉시 동작하고 운영 데이터로 남는다

### 에픽 I. 관리자 도구

목표:

- 대표 1인이 핵심 운영을 수행할 수 있어야 한다

태스크:

- P0 관리자 유저 목록 조회
- P0 신고 목록 조회
- P0 포인트 거래 내역 조회
- P0 게시글 숨김 처리
- P0 유저 정지 처리
- P0 DM 요청 상태 조회
- P1 관리자 액션 로그 저장
- P1 검색 및 필터 기능

완료 기준:

- 관리자 페이지에서 최소한의 제재와 조회가 가능하다

2. 차단 시 환불 여부
3. 메시지 보관 기간
4. 탈퇴 후 보존 로그 범위
5. 신고 누적 정지 기준
