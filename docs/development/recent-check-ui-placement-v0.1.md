# 최근검사 확인 UI Placement v0.1

## 목적
- `최근검사 확인` 기능을 실제 화면 어디에, 어떤 강도로 노출할지 정리한다.

## 원칙
- 의료 배지처럼 과도하게 보이지 않게 한다.
- 신뢰 정보 중 하나로 다룬다.
- 사용자에게 `현재 안전 보장`으로 읽히지 않게 한다.
- 리스트보다 프로필 상세에서 더 많은 정보를 제공한다.

## 1. 프로필 상세

### 위치
- 닉네임/기본 정보 아래
- 자기소개 섹션 위

### 섹션명
- `신뢰 정보`

### 구성
- 배지 칩: `최근검사 확인`
- 검사일
- 표시 만료일
- 클릭 시 상세 모달

### 예시 구조
1. 닉네임
2. 나이/지역/역할
3. 신뢰 정보
4. 최근검사 확인 칩 + 날짜
5. 자기소개
6. 최근 게시글
7. DM CTA

### 이유
- 프로필의 중심은 여전히 사람 자체여야 한다.
- 배지는 보조적 신뢰 신호여야 한다.

## 2. 프로필 리스트/피드 카드

### 위치
- 닉네임 옆 또는 하단 메타 영역

### 구성
- 작은 칩 하나만 노출
- 문구: `최근검사 확인`

### 생략할 정보
- 검사일
- 유효기간
- 상세 설명

### 이유
- 카드 리스트에서 정보가 과해지면 계급장처럼 보인다.

## 3. DM 요청 바텀시트

### 위치
- 상대 요약 카드 아래
- 비용/환불 규칙 위

### 구성
- `최근검사 확인 정보 공개 중`
- 검사일
- 작은 안내 문구

### 역할
- 과금 결정을 앞둔 순간의 신뢰 보조 신호

## 4. 배지 상세 모달

### 열리는 위치
- 프로필 상세의 배지 클릭
- DM 요청 바텀시트의 정보 클릭

### 구성
- 제목: `최근검사 확인`
- 설명: `최근 STI 검사 확인 정보를 공개한 프로필입니다`
- 경고 문구: `검사 시점 기준 정보이며 현재 상태를 보증하지 않습니다`

## 5. 마이페이지/프로필 편집

### 위치
- 프로필 편집 내부의 `신뢰 정보` 섹션

### 구성
- 현재 상태
- 공개 여부 토글
- 제출/재제출/철회 버튼

## 6. 관리자 화면

### 위치
- 기존 `/admin` 하위 STI 섹션

### 구성
- 대기열
- 제출 상세
- 승인/반려
- 감사 로그

## 7. 비권장 배치
- 홈 화면 상단 대배너
- 닉네임 앞 뱃지 고정 노출
- 카드 썸네일 위 강한 라벨
- 프로필 1순위 정보처럼 보이게 하는 구조

## 8. 최종 권장안
- 프로필 상세: 정보형 배지 + 날짜 + 상세 모달
- 리스트: 작은 칩만
- DM 요청 시트: 보조 신뢰 정보로 한 번 더 노출
