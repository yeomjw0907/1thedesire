# 욕망백서 크론 / 배치 작업 문서 v0.1

## 1. 목적

- 예약 실행이 필요한 백그라운드 작업을 정리한다.
- 만료, 환불, 알림 같은 자동 작업의 기준을 문서화한다.

## 2. 핵심 작업 목록

### 2-1. DM 만료 처리

주기:

- 5분 또는 10분 간격 권장

조건:

- `chat_rooms.status = pending`
- `request_expires_at <= now()`

처리:

- status를 expired로 변경
- 90P 환불
- consent event 기록
- point transaction 기록

### 2-2. 알림 큐 처리

주기:

- 1분 또는 5분 간격

대상:

- 요청 도착
- 요청 수락
- 거절 / 만료 / 환불

### 2-3. 운영 요약 집계

주기:

- 하루 1회

대상:

- 가입 수
- DM 요청 수
- 거절 / 만료 / 환불
- 신고 / 차단

### 2-4. 메시지 보관 기간 관리

주기:

- 하루 1회

대상:

- 90일 경과 메시지 처리 정책에 따라 아카이브 또는 비노출 검토

## 3. 구현 원칙

- 크론 작업은 idempotent 해야 한다
- 같은 room에 환불이 중복 반영되면 안 된다
- 실패 로그를 반드시 남긴다

## 4. 최종 판단

욕망백서의 배치 작업 핵심은 `만료와 환불의 자동 일관성`이다.
