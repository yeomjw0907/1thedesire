너는 현재 Next.js 15 + React 19 + Supabase 기반 프로젝트에서 작업하는 시니어 엔지니어다.

이번 작업 목표는 서비스 구조를 `홈 중심 게시글 탐색 구조`로 재편하는 것이다.

2. docs/planning/home-centered-page-structure-v0.1.md
3. docs/planning/screens-v0.1.md
4. docs/planning/wireframes-v0.1.md
5. docs/planning/point-monetization-ux-v0.1.md
6. docs/development/recent-check-ui-placement-v0.1.md
7. docs/development/recent-check-copy-system-v0.1.md

핵심 방향:
- 홈은 게시글 메인 피드이자 탐색 허브다
- 검색 페이지 기능은 홈으로 흡수한다
- 검색 페이지는 제거 대상으로 본다
- 게시글 카드의 `프로필 보기` 버튼은 제거한다
- 프로필 이미지 또는 닉네임 클릭으로 프로필 상세에 진입하게 한다
- 게시글 카드에는 `DM 보내기` 버튼을 둔다
- DM 거절 기능은 제거한다
- DM 목록 구조는 `전체 / 읽지 않음 / 요청`으로 바꾼다
- 프로필은 이미지, 상세 성향, 최근검사 확인, 신뢰 정보가 보이게 강화한다
- 포인트 화면 계좌 정보는 아래로 교체한다
  - 우리은행
  - 1005-90-369058
  - 주식회사 98점7도

중요:
- 이번 작업은 기존 서비스의 중심축을 바꾸는 작업이다
- 검색 기능을 홈에 흡수하되, 사용자가 갑자기 길을 잃지 않게 해야 한다
- 법률성 긴 문구는 결제 화면 본문에 전부 노출하지 말고, 핵심 안내와 상세 안내를 분리하라
- 최근검사 확인 UI는 이미 만든 문서와 컴포넌트 방향을 유지하라
- 과장된 카피는 쓰지 마라
- 기존 한국어 문자열 깨짐이 있는 파일은 함부로 더 확산시키지 말고, 수정 범위를 최소화해서 다뤄라

구현 범위:

1. 홈 개편
- 대상 파일:
  - src/app/(main)/home/page.tsx
  - 관련 카드 컴포넌트가 분리 필요하면 src/components/post/* 또는 적절한 위치에 분리
- 해야 할 일:
  - 홈을 게시글 메인 페이지로 유지
  - 상단에 글쓰기 버튼과 새로고침 버튼 추가
  - 검색 페이지 기능 일부를 홈 필터로 가져오기
  - 필터 예시:
    - 최신순
    - 인기순
    - 성향
    - 성별
    - 최근검사 확인
    - 인증된 사용자
  - 게시글 카드에 사진 노출 지원 유지 또는 개선
  - 게시글 카드에서 프로필 이미지/닉네임 클릭 시 프로필 이동
  - `프로필 보기` 버튼 제거
  - `DM 보내기` 버튼 추가

2. 검색 페이지 정리
- 대상 파일:
  - src/app/(main)/search/page.tsx
  - src/components/layout/BottomNav.tsx
- 해야 할 일:
  - 검색 탭 제거 방향으로 반영
  - 홈으로 기능이 흡수되었음을 기준으로 네비게이션 구조 정리
  - 삭제가 부담되면 우선 리다이렉트 또는 숨김 처리 후 최소 잔재만 남겨라

3. DM 개편
- 대상 파일:
  - src/app/(main)/dm/page.tsx
  - 관련 상태/액션 파일
  - src/components/dm/*
  - src/lib/actions/dm.ts
  - 관련 타입 파일
- 해야 할 일:
  - 탭을 `전체 / 읽지 않음 / 요청`으로 변경
  - `거절` 기능 제거
  - 거절 관련 UI/서버 액션/문구/환불 로직 점검
  - 가능하다면 읽음 여부를 반영할 수 있는 구조로 설계
  - `요청`은 나에게 온 신규 요청 중심으로 보이게 정리

4. 포인트 화면 개편
- 대상 파일:
  - src/app/(main)/points/page.tsx
  - src/components/points/ChargeSheet.tsx
- 계좌 정보 교체:
  - 은행: 우리은행
  - 계좌번호: 1005-90-369058
  - 예금주: 주식회사 98점7도
- UX 방향:
  - 상단 핵심 안내만 짧게 노출
  - 긴 안내사항은 접기/더보기/별도 정책 링크로 분리
  - 아래 내용은 요약/분리해서 반영:
    - 결제 후 1~3분 내 자동 충전
    - 소액 충전 권장
    - 입금자명 정확 입력
    - 입금자명 제한 규칙
    - 충전 오류 시 이메일 문의
    - 새로고침 안내
    - 포인트 사용처 안내
    - 대화 자동 삭제 가능성 안내
    - 유료서비스/환불/정책 링크

5. 프로필 강화
- 대상 파일:
  - src/app/(main)/profile/[id]/page.tsx
  - src/app/(main)/profile/page.tsx
  - src/app/(main)/profile/edit/page.tsx
  - src/components/profile/*
  - src/types/*
- 해야 할 일:
  - 프로필 이미지 영역 추가
  - 상세 성향 노출 영역 추가
  - 최근검사 확인 배지 위치 재점검
  - 신뢰 정보 섹션 정리
  - 최근 게시글과 자기소개 구조를 더 명확히 분리

작업 순서:
1. 현재 구조 분석
2. 홈/검색/DM/포인트/프로필 영향 범위 정리
3. 홈 개편
4. 검색 제거 또는 홈 통합 반영
5. DM 구조 개편
6. 포인트 화면 개편
7. 프로필 강화
8. lint/tsc 실행
9. 변경 사항과 남은 리스크 보고
